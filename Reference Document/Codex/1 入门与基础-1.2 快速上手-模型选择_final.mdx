# Codex 模型

## 推荐模型

*Image Description: Codex 应用显示模型选择界面，展示 gpt-5.3-codex、gpt-5.2-codex 和 gpt-5.1-codex-mini 三个模型的卡片式布局，每个卡片包含能力等级（五星）、速度等级（四星）、支持的平台图标和描述信息。*

对于 Codex 中的大多数编码任务，从 `gpt-5.3-codex` 开始。它适用于 ChatGPT 身份验证的 Codex 会话，包括 Codex 应用、CLI、IDE 扩展和 Codex 云。该模型的 API 访问即将推出。

## 替代模型

*Image Description: Codex 应用显示可折叠的模型卡片列表，包括 gpt-5.2-codex、gpt-5.1-codex-max、gpt-5.2、gpt-5.1、gpt-5.1-codex、gpt-5-codex、gpt-5-codex-mini 和 gpt-5。每个卡片可以展开显示详细的功能特性列表。*

Codex 在列出的模型中效果最佳。

您还可以将 Codex 指向任何支持[聊天补全](https://platform.openai.com/docs/api-reference/chat)或[响应 API](https://platform.openai.com/docs/api-reference/responses)的模型和提供商，以适应您的特定用例。

<Callout type="warning">
对聊天补全 API 的支持已被弃用，将在 Codex 的未来版本中移除。
</Callout>

## 配置模型

### 配置您的默认本地模型

Codex CLI 和 IDE 扩展使用相同的 `config.toml` [配置文件](https://developers.openai.com/codex/config-basic)。要指定模型，请在配置文件中添加 `model` 条目。如果您未指定模型，Codex 应用、CLI 或 IDE 扩展将默认为推荐模型。

```toml
model = "gpt-5.2"
```

### 临时选择不同的本地模型

在 Codex CLI 中，您可以在活动线程中使用 `/model` 命令来更改模型。在 IDE 扩展中，您可以使用输入框下方的模型选择器来选择模型。

要使用特定模型启动新的 Codex CLI 线程或为 `codex exec` 指定模型，可以使用 `--model`/`-m` 标志：

```bash
codex -m gpt-5.3-codex
```

### 为云端任务选择模型

目前，您无法更改 Codex 云任务的默认模型。
