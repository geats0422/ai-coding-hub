# Codex CLI 斜杠命令

斜杠命令让您可以快速、键盘优先地控制 Codex。在编辑器中键入 `/` 打开斜杠弹出窗口，选择一个命令，Codex 将执行诸如切换模型、调整权限或总结长对话等操作，而无需离开终端。

本指南向您展示如何：

- 为任务找到正确的内置斜杠命令
- 使用 `/model`、`/personality`、`/permissions` 和 `/status` 等命令引导活动会话

## 内置斜杠命令

Codex 附带以下命令。打开斜杠弹出窗口并开始键入命令名称以过滤列表。

| 命令 | 目的 | 使用时机 |
| ---- | ---- | -------- |
| `/permissions` | 设置 Codex 可以在不先询问的情况下执行的操作。 | 在会话中放松或收紧批准要求，例如在自动和只读之间切换。 |
| `/apps` | 浏览应用（连接器）并将它们插入到您的提示词中。 | 在要求 Codex 使用它之前，将应用附加为 `$app-slug`。 |
| `/compact` | 总结可见对话以释放令牌。 | 在长时间运行后使用，以便 Codex 保留关键点而不耗尽上下文窗口。 |
| `/diff` | 显示 Git 差异，包括 Git 尚未跟踪的文件。 | 在提交或运行测试之前审查 Codex 的编辑。 |
| `/exit` | 退出 CLI（与 `/quit` 相同）。 | 替代拼写；两个命令都会退出会话。 |
| `/feedback` | 向 Codex 维护者发送日志。 | 报告问题或与支持共享诊断信息。 |
| `/init` | 在当前目录生成 `AGENTS.md` 脚手架。 | 捕获您正在工作的存储库或子目录的持久指令。 |
| `/logout` | 从 Codex 退出。 | 在共享机器上清除本地凭据。 |
| `/mcp` | 列出配置的模型上下文协议（MCP）工具。 | 检查 Codex 可以在会话期间调用哪些外部工具。 |
| `/mention` | 将文件附加到对话中。 | 指向您希望 Codex 接下来检查的特定文件或文件夹。 |
| `/model` | 选择活动模型（以及可用时的推理工作）。 | 在运行任务之前，在通用模型（如 `gpt-4.1-mini`）和更深入的推理模型之间切换。 |
| `/plan` | 切换到计划模式，并可选择发送提示词。 | 要求 Codex 在实施工作开始之前提出执行计划。 |
| `/personality` | 为响应选择沟通风格。 | 使 Codex 更简洁、更具解释性或更具协作性，而无需更改您的指令。 |
| `/ps` | 显示实验性后台终端及其最近输出。 | 检查长时间运行的命令，而无需离开主记录。 |
| `/fork` | 将当前对话分叉到新线程中。 | 将活动会话分支以探索新方法，而不会丢失当前记录。 |
| `/resume` | 从您的会话列表恢复保存的对话。 | 从之前的 CLI 会话继续工作，而无需重新开始。 |
| `/new` | 在同一个 CLI 会话中开始新的对话。 | 当您想要在同一个代码库中获得新的提示词时重置聊天上下文，而无需离开 CLI。 |
| `/quit` | 退出 CLI。 | 立即离开会话。 |
| `/review` | 要求 Codex 审查您的工作树。 | 在 Codex 完成工作后或在您想要第二双眼睛查看本地更改时运行。 |
| `/status` | 显示会话配置和令牌使用情况。 | 确认活动模型、批准策略、可写根目录和剩余上下文容量。 |
| `/debug-config` | 打印配置层和需求诊断。 | 调试优先级和策略要求，包括实验性网络约束。 |
| `/statusline` | 交互式配置页脚字段。 | 选择和重新排序页脚项目（模型/上下文/限制/git/令牌/会话）并持久化到 config.toml 中。 |

<Callout type="note">
`/quit` 和 `/exit` 都会退出 CLI。仅在保存或提交任何重要工作后使用它们。
</Callout>

<Callout type="note">
`/approvals` 命令仍然可以作为别名使用，但它不再出现在斜杠弹出窗口列表中。
</Callout>

## 使用斜杠命令控制您的会话

以下工作流可以让您的会话保持正轨，而无需重启 Codex。

### 使用 `/model` 设置活动模型

1. 启动 Codex 并打开编辑器。
2. 键入 `/model` 并按 Enter。
3. 从弹出窗口中选择一个模型，如 `gpt-4.1-mini` 或 `gpt-4.1`。

预期结果：Codex 在记录中确认新模型。运行 `/status` 以验证更改。

### 使用 `/personality` 设置沟通风格

使用 `/personality` 更改 Codex 的沟通方式，而无需重写您的提示词。

1. 在活动对话中，键入 `/personality` 并按 Enter。
2. 从弹出窗口中选择一种风格。

预期结果：Codex 在记录中确认新风格，并在会话中的后续响应中使用它。

Codex 支持 `friendly`、`pragmatic` 和 `none` 人格。使用 `none` 禁用个性化指令。

如果活动模型不支持特定于人格的指令，Codex 会隐藏此命令。

### 使用 `/plan` 切换到计划模式

1. 键入 `/plan` 并按 Enter，将活动对话切换到计划模式。
2. 可选：提供内联提示词文本（例如，`/plan Propose a migration plan for this service`）。
3. 您可以在使用内联 `/plan` 参数时粘贴内容或附加图像。

预期结果：Codex 进入计划模式，并使用您可选的内联提示词作为第一个规划请求。

当任务已经在运行时，`/plan` 暂时不可用。

### 使用 `/permissions` 更新权限

1. 键入 `/permissions` 并按 Enter。
2. 选择与您的舒适度相匹配的批准预设，例如 `Auto` 表示无人值守运行，或 `Read Only` 以审查编辑。

预期结果：Codex 宣布更新后的策略。未来的操作尊重新的批准模式，直到您再次更改它。

### 使用 `/status` 检查会话

1. 在任何对话中，键入 `/status`。
2. 查看输出以了解活动模型、批准策略、可写根目录和当前令牌使用情况。

预期结果：您会看到类似于 `codex status` 在 shell 中打印的内容的摘要，确认 Codex 正在您期望的位置运行。

### 使用 `/debug-config` 检查配置层

1. 键入 `/debug-config`。
2. 查看输出以了解配置层顺序（从最低优先级开始）、开/关状态和策略来源。

预期结果：Codex 打印层诊断加上策略详情，如 `allowed_approval_policies`、`allowed_sandbox_modes`、`mcp_servers`、`rules`、`enforce_residency` 和 `experimental_network`（在配置时）。

使用此输出来调试为什么有效设置与 `config.toml` 不同。

### 使用 `/statusline` 配置页脚项目

1. 键入 `/statusline`。
2. 使用选择器切换和重新排序项目，然后确认。

预期结果：页脚状态行立即更新并持久化到 `config.toml` 中的 `tui.status_line`。

可用的状态行项目包括模型、模型+推理、上下文统计、速率限制、git 分支、令牌计数器、会话 ID、当前目录/项目根目录和 Codex 版本。

### 使用 `/ps` 检查后台终端

1. 键入 `/ps`。
2. 查看后台终端列表及其状态。

预期结果：Codex 显示每个后台终端的命令加上最多三个最近的、非空的输出行，以便您可以快速查看进度。

当 `unified_exec` 在使用时，会出现后台终端；否则列表可能为空。

### 使用 `/compact` 保持记录精简

1. 在长时间交换后，键入 `/compact`。
2. 当 Codex 提供总结迄今为止的对话时确认。

预期结果：Codex 用简明摘要替换先前的轮次，释放上下文同时保留关键细节。

### 使用 `/diff` 审查更改

1. 键入 `/diff` 以检查 Git 差异。
2. 在 CLI 内滚动输出以审查编辑和添加的文件。

预期结果：Codex 显示您已暂存的更改、尚未暂存的更改以及 Git 尚未开始跟踪的文件，以便您决定保留什么。

### 使用 `/mention` 高亮文件

1. 键入 `/mention` 后跟一个路径，例如 `/mention src/lib/api.ts`。
2. 从弹出窗口中选择匹配的结果。

预期结果：Codex 将文件添加到对话中，确保后续轮次直接引用它。

### 使用 `/new` 开始新的对话

1. 键入 `/new` 并按 Enter。

预期结果：Codex 在同一个 CLI 会话中开始新的对话，因此您可以切换任务而无需离开终端。

### 使用 `/resume` 恢复保存的对话

1. 键入 `/resume` 并按 Enter。
2. 从保存会话选择器中选择您想要的会话。

预期结果：Codex 重新加载所选对话的记录，以便您可以从上次中断的地方继续，保持原始历史完整。

### 使用 `/fork` 分叉当前对话

1. 键入 `/fork` 并按 Enter。

预期结果：Codex 将当前对话克隆到一个带有新 ID 的新线程中，保留原始记录不变，以便您可以并行探索替代方法。

如果您想要分叉保存的会话而不是当前的，运行 `codex fork` 在您的终端中打开会话选择器。

### 使用 `/init` 生成 `AGENTS.md`

1. 在您希望 Codex 查找持久指令的目录中运行 `/init`。
2. 查看生成的 `AGENTS.md`，然后根据您的存储库约定进行编辑。

预期结果：Codex 创建一个 `AGENTS.md` 脚手架，您可以对其进行完善并提交以用于未来的会话。

### 使用 `/review` 请求工作树审查

1. 键入 `/review`。
2. 如果您想要检查确切的文件更改，请使用 `/diff` 进行后续操作。

预期结果：Codex 总结它在您的工作树中发现的问题，专注于行为更改和缺失的测试。除非您在 `config.toml` 中设置了 `review_model`，否则它使用当前会话模型。

### 使用 `/mcp` 列出 MCP 工具

1. 键入 `/mcp`。
2. 查看列表以确认哪些 MCP 服务器和工具可用。

预期结果：您可以看到配置的模型上下文协议（MCP）工具，Codex 可以在此会话中调用。

### 使用 `/apps` 浏览应用

1. 键入 `/apps`。
2. 从列表中选择一个应用。

预期结果：Codex 将应用提及作为 `$app-slug` 插入到编辑器中，因此您可以立即要求 Codex 使用它。

### 使用 `/feedback` 发送反馈

1. 键入 `/feedback` 并按 Enter。
2. 按照提示包括日志或诊断信息。

预期结果：Codex 收集请求的诊断信息并将其提交给维护者。

### 使用 `/logout` 退出

1. 键入 `/logout` 并按 Enter。

预期结果：Codex 清除当前用户会话的本地凭据。

### 使用 `/quit` 或 `/exit` 退出 CLI

1. 键入 `/quit`（或 `/exit`）并按 Enter。

预期结果：Codex 立即退出。首先保存或提交任何重要工作。
