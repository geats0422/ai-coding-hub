# Codex IDE 扩展

Codex 是 OpenAI 的编码代理，可以读取、编辑和运行代码。它可以帮助您更快地构建、修复错误并理解不熟悉的代码。通过 Codex VS Code 扩展，您可以在 IDE 中并排使用 Codex，或将任务委派给 Codex 云。

ChatGPT Plus、Pro、Business、Edu 和 Enterprise 计划包含 Codex。了解[包含的内容](https://developers.openai.com/codex/pricing)。

## 扩展设置

Codex IDE 扩展适用于 VS Code 分支，如 Cursor 和 Windsurf。

您可以从 [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=openai.chatgpt) 获取 Codex 扩展，或为您的 IDE 下载：

- [下载 Visual Studio Code](vscode:extension/openai.chatgpt)
- [下载 Cursor](cursor:extension/openai.chatgpt)
- [下载 Windsurf](windsurf:extension/openai.chatgpt)
- [下载 Visual Studio Code Insiders](https://marketplace.visualstudio.com/items?itemName=openai.chatgpt)
- [下载 JetBrains IDE](#jetbrains-ide-integration)

Codex VS Code 扩展在 macOS 和 Linux 上可用。Windows 支持是实验性的。要获得最佳 Windows 体验，请在 WSL 工作区中使用 Codex，并遵循我们的 [Windows 设置指南](/codex/windows)。

安装后，您会在左侧边栏中看到扩展与其他扩展在一起。如果使用 VS Code，如果没立即看到 Codex，请重新启动编辑器。

如果使用 Cursor，活动栏默认水平显示。折叠的项目可能会隐藏 Codex，因此您可以固定它并重新排列扩展的顺序。

*Image Description: VS Code 侧边栏显示 Codex 扩展图标和已安装扩展列表的位置示意图。*

## JetBrains IDE 集成

如果您想在 Rider、IntelliJ、PyCharm 或 WebStorm 等 JetBrains IDE 中使用 Codex，请安装 JetBrains IDE 集成。它支持使用 ChatGPT、API 密钥或 JetBrains AI 订阅登录。

### 将 Codex 移动到右侧边栏

在 VS Code 中，您可以拖动 Codex 图标到编辑器的右侧以将其移动到右侧边栏。

在某些 IDE（如 Cursor）中，您可能需要先临时更改活动栏方向：

1. 打开编辑器设置并搜索 `activity bar`（在工作区设置中）。
2. 将方向更改为 `vertical`。
3. 重新启动编辑器。

![codex-workbench-setting](https://cdn.openai.com/devhub/docs/codex-workbench-setting.webp)

现在将 Codex 图标拖动到右侧边栏（例如，在 Cursor 聊天旁边）。Codex 作为另一个标签页出现在侧边栏中。

移动后，将活动栏方向重置为 `horizontal` 以恢复默认行为。

## 登录

安装扩展后，它会提示您使用 ChatGPT 账户或 API 密钥登录。您的 ChatGPT 计划包含使用额度，因此您无需额外设置即可使用 Codex。在[定价页面](https://developers.openai.com/codex/pricing)了解更多。

## 更新扩展

扩展会自动更新，但您也可以在编辑器中打开扩展页面以检查更新。

## 设置键盘快捷键

Codex 包含您可以在 IDE 设置中绑定为键盘快捷键的命令（例如，切换 Codex 聊天或将项目添加到 Codex 上下文）。

要查看所有可用的命令并将它们绑定为键盘快捷键，请在 Codex 聊天中选择设置图标，然后选择 **Keyboard shortcuts**。

您也可以参阅 [Codex IDE 扩展命令](https://developers.openai.com/codex/ide/commands) 页面。

有关支持的斜杠命令列表，请参阅 [Codex IDE 扩展斜杠命令](https://developers.openai.com/codex/ide/slash-commands)。

## 使用 Codex IDE 扩展

### 使用编辑器上下文进行提示

在编辑器中使用 Codex 进行聊天、编辑和预览更改。当 Codex 从打开的文件和选定的代码中获取上下文时，您可以编写更短的提示词并获得更快、更相关的结果。

您可以在提示词中通过这样标记来引用编辑器中的任何文件：

```
Use @example.tsx as a reference to add a new page named "Resources" to the app that contains a list of resources defined in @resources.ts
```

### 切换模型

您可以使用输入框下方的选择器切换模型。

*Image Description: Codex IDE 扩展中的模型选择器，显示下拉菜单包含可用的模型选项。*

### 调整推理工作

您可以调整推理工作以控制 Codex 在响应之前思考的时间。较高的努力可以帮助复杂任务，但响应需要更长时间。较高的努力也会使用更多令牌，并可能更快地消耗您的速率限制（尤其是使用 GPT-5-Codex 时）。

使用上面显示的相同模型选择器，并为每个模型选择 `low`、`medium` 或 `high`。从 `medium` 开始，只有在需要更多深度时才切换到 `high`。

### 选择批准模式

默认情况下，Codex 以 `Agent` 模式运行。在此模式下，Codex 可以自动在工作目录中读取文件、进行编辑和运行命令。Codex 仍然需要在工作目录之外工作或访问网络之前获得您的批准。

当您只想聊天，或者您想在进行更改之前进行规划时，使用输入框下方的选择器切换到 `Chat`。

*Image Description: Codex IDE 扩展中的批准模式选择器，显示 Chat、Agent 和 Agent (Full Access) 三个选项。*

如果您需要 Codex 读取文件、进行编辑并在没有批准的情况下运行具有网络访问权限的命令，请使用 `Agent (Full Access)`。在执行此操作之前请谨慎。

### 委派到云端

您可以将更大的作业卸载到云端的 Codex，然后跟踪进度并在不离开 IDE 的情况下审查结果。

1. 为 Codex 设置[云环境](https://chatgpt.com/codex/settings/environments)。
2. 选择您的环境并选择 **Run in the cloud**。

您可以让 Codex 从 `main` 运行（对于开始新想法很有用），或从您的本地更改运行（对于完成任务很有用）。

*Image Description: 从 IDE 启动云任务的界面，显示环境选择和运行选项。*

当您从本地对话启动云任务时，Codex 会记住对话上下文，以便它可以从上次中断的地方继续。

### 云任务后续操作

Codex 扩展使预览云端更改变得简单。您可以要求后续在云端运行，但通常您会希望将更改本地应用以进行测试和完成。当您在本地继续对话时，Codex 也会保留上下文以节省您的时间。

*Image Description: 将云任务加载到 IDE 中的界面，显示预览和应用选项。*

您也可以在 [Codex 云界面](https://chatgpt.com/codex)中查看云任务。

### 网页搜索

Codex 附带了一个官方的网页搜索工具。对于 Codex IDE 扩展中的本地任务，Codex 默认启用网页搜索并从网页搜索缓存中提供结果。缓存是 OpenAI 维护的网页结果索引，因此缓存模式返回预索引的结果而不是获取实时页面。这减少了来自任意实时内容的提示注入暴露，但您仍应将网页结果视为不受信任的。如果您将沙箱配置为[完全访问](https://developers.openai.com/codex/security)，网页搜索默认为实时结果。请参阅[基本配置](https://developers.openai.com/codex/config-basic)以禁用网页搜索或切换到获取最新数据的实时结果。

每当 Codex 查找某些内容时，您会在记录或 `codex exec --json` 输出中看到 `web_search` 项目。

### 将图像拖放到提示词中

您可以将图像拖放到提示词编辑器中以将它们作为上下文包含。

在拖放图像时按住 Shift。VS Code 会阻止扩展接受拖放。
