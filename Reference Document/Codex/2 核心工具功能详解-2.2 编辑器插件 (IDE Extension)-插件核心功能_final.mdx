# Codex IDE 扩展功能

Codex IDE 扩展让您可以在 VS Code、Cursor、Windsurf 和其他 VS Code 兼容编辑器中直接访问 Codex。它使用与 Codex CLI 相同的代理，并共享相同的配置。

<AdPlaceholder />

## 提示词 Codex

在编辑器中使用 Codex 进行聊天、编辑和预览更改。当 Codex 从打开的文件和选定的代码中获取上下文时，您可以编写更短的提示词并获得更快、更相关的结果。

您可以在提示词中通过这样标记来引用编辑器中的任何文件：

```
Use @example.tsx as a reference to add a new page named "Resources" to the app that contains a list of resources defined in @resources.ts
```

您可以使用选择器切换模型。

## 调整推理工作

您可以调整推理工作以控制 Codex 在响应之前思考的时间。较高的努力可以帮助复杂任务，但响应需要更长时间。较高的努力也会使用更多令牌，并可能更快地消耗您的速率限制（尤其是使用 GPT-5-Codex 时）。

使用上面显示的相同模型选择器，并为每个模型选择 `low`、`medium` 或 `high`。从 `medium` 开始，只有在需要更多深度时才切换到 `high`。

## 选择批准模式

默认情况下，Codex 以 `Agent` 模式运行。在此模式下，Codex 可以自动在工作目录中读取文件、进行编辑和运行命令。Codex 仍然需要在工作目录之外工作或访问网络之前获得您的批准。

当您只想聊天，或者您想在进行更改之前进行规划时，使用输入框下方的选择器切换到 `Chat`。

*Image Description: Codex IDE 扩展中的批准模式选择器，显示 Agent、Chat 和 Agent (Full Access) 模式选项。*

如果您需要 Codex 读取文件、进行编辑并在没有批准的情况下运行具有网络访问权限的命令，请使用 `Agent (Full Access)`。在执行此操作之前请谨慎。

## 云端委派

您可以将更大的作业卸载到云端的 Codex，然后跟踪进度并在不离开 IDE 的情况下审查结果。

1. 为 Codex 设置[云环境](https://chatgpt.com/codex/settings/environments)。
2. 选择您的环境并选择 **Run in the cloud**。

您可以让 Codex 从 `main` 运行（对于开始新想法很有用），或从您的本地更改运行（对于完成任务很有用）。

*Image Description: 从 IDE 启动云端任务的界面，显示环境选择和运行按钮。*

当您从本地对话启动云任务时，Codex 会记住对话上下文，以便它可以从上次中断的地方继续。

## 云端任务后续操作

Codex 扩展使预览云端更改变得简单。您可以要求后续在云端运行，但通常您会希望将更改本地应用以测试和完成。当您在本地继续对话时，Codex 也会保留上下文以节省您的时间。

*Image Description: 将云端任务加载到 IDE 中的界面，显示预览、应用和继续对话选项。*

您也可以在 [Codex 云界面](https://chatgpt.com/codex)中查看云端任务。

## 网页搜索

Codex 附带了一个官方的网页搜索工具。对于 Codex IDE 扩展中的本地任务，Codex 默认启用网页搜索并从网页搜索缓存中提供结果。缓存是 OpenAI 维护的网页结果索引，因此缓存模式返回预索引的结果而不是获取实时页面。这减少了来自任意实时内容的提示注入暴露，但您仍应将网页结果视为不受信任的。如果您将沙箱配置为[完全访问](https://developers.openai.com/codex/security)，网页搜索默认为实时结果。请参阅[基本配置](https://developers.openai.com/codex/config-basic)以禁用网页搜索或切换到获取最新数据的实时结果。

每当 Codex 查找某些内容时，您会在记录或 `codex exec --json` 输出中看到 `web_search` 项目。

## 将图像拖放到提示词中

您可以将图像拖放到提示词编辑器中以将它们作为上下文包含。在拖放图像时按住 Shift。VS Code 会阻止扩展接受拖放。
