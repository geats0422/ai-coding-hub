# Codex IDE 扩展设置

使用这些设置来自定义 Codex IDE 扩展。

<AdPlaceholder />

## 更改设置

要更改设置，请按照以下步骤操作：

1. 打开编辑器设置。
2. 搜索 `Codex` 或设置名称。
3. 更新值。

Codex IDE 扩展使用 Codex CLI。某些行为（如默认模型、批准和沙箱设置）在共享的 `~/.codex/config.toml` 文件中配置，而不是在编辑器设置中。请参阅[基本配置](https://developers.openai.com/codex/config-basic)。

## 设置参考

| 设置 | 描述 |
| ---- | ---- |
| `chatgpt.cliExecutable` | 仅限开发：到 Codex CLI 可执行文件的路径。除非您正在积极开发 Codex CLI，否则不需要设置此选项。如果您手动设置此选项，扩展的某些部分可能无法按预期工作。 |
| `chatgpt.commentCodeLensEnabled` | 在待办事项注释上方显示 CodeLens，以便您可以使用 Codex 完成它们。 |
| `chatgpt.localeOverride` | Codex UI 的首选语言。留空以自动检测。 |
| `chatgpt.openOnStartup` | 扩展完成启动时聚焦 Codex 侧边栏。 |
| `chatgpt.runCodexInWindowsSubsystemForLinux` | 仅限 Windows：当 Windows 子系统 for Linux (WSL) 可用时在 WSL 中运行 Codex。建议用于改进沙箱安全性和更好的性能。Windows 上的 Codex 代理模式当前需要 WSL。更改此设置会重新加载 VS Code 以应用更改。 |
