# Codex 应用

Codex 应用是一个专注于桌面的体验，用于并行处理 Codex 线程，具有内置的工作树支持、自动化和 Git 功能。

ChatGPT Plus、Pro、Business、Edu 和 Enterprise 计划包含 Codex。了解[包含的内容](https://developers.openai.com/codex/pricing)。

*Image Description: Codex 应用窗口显示项目侧边栏、活动线程和审查面板的界面布局。*

## 入门

Codex 应用目前在 macOS（Apple Silicon）上可用。

<Callout type="step">
1. 下载并安装 Codex 应用

    Codex 应用目前仅适用于 macOS。

    <Callout type="download">
    <a href="https://persistent.oaistatic.com/codex-app-prod/Codex.dmg" target="_blank">下载 macOS 版</a>
    </Callout>

    <Callout type="note">
    <a href="https://openai.com/form/codex-app/" target="_blank">注册接收 Windows 和 Linux 版通知</a>
    </Callout>

2. 打开 Codex 并登录

    下载并安装 Codex 应用后，打开它并使用您的 ChatGPT 账户或 OpenAI API 密钥登录。

    如果您使用 OpenAI API 登录，某些功能（如[云线程](https://developers.openai.com/codex/prompting#threads)）可能不可用。

3. 选择项目

    选择您希望 Codex 工作的项目文件夹。

    如果您之前使用过 Codex 应用、CLI 或 IDE 扩展，您会看到您曾经处理过的过去项目。

4. 发送您的第一条消息

    选择项目后，确保选中 **Local** 以让 Codex 在您的机器上运行，然后向 Codex 发送您的第一条消息。

    您可以向 Codex 询问关于项目或您计算机的任何问题。以下是一些示例：

    - "Tell me about this project"
    - "Build a classic Snake game in this repo."
    - "Find and fix bugs in my codebase with minimal, high-confidence changes."

    如果您需要更多灵感，请查看[探索部分](https://developers.openai.com/codex/explore)。
</Callout>

## 使用 Codex 应用

### 跨项目多任务处理

使用一个 Codex 应用窗口跨项目运行任务。为每个代码库添加一个项目，并根据需要切换它们。

如果您使用过 [Codex CLI](https://developers.openai.com/codex/cli)，项目就像在特定目录中启动会话一样。

如果您在具有两个或更多应用程序或包的单个存储库中工作，请将不同的项目拆分到单独的应用项目中，以便[沙箱](https://developers.openai.com/codex/security)只包含该项目的文件。

### 内置 Git 工具

Codex 应用在应用内提供常见的 Git 功能。

差异面板显示您本地项目或工作树检出中更改的 Git 差异。您还可以为 Codex 添加内联评论，并暂存或回滚特定块或整个文件。

您还可以直接从 Codex 应用对本地和工作树任务进行提交、推送和创建拉取请求。

有关更高级的 Git 任务，请使用[集成终端](#集成终端)。

### 工作树支持

创建新线程时，选择 **Local** 或 **Worktree**。**Local** 直接在您的项目中工作。**Worktree** 创建一个新的 [Git 工作树](https://git-scm.com/docs/git-worktree)，以便更改与您常规项目保持隔离。

当您想要尝试新想法而不触及当前工作，或者您希望 Codex 在同一个项目中并行运行独立任务时，请使用 **Worktree**。

自动化在 Git 存储库的专用后台工作树中运行，并在非版本控制项目中直接在工作目录中运行。

[详细了解如何在 Codex 应用中使用工作树。](https://developers.openai.com/codex/app/worktrees)

### 集成终端

每个线程都包含一个内置终端，作用域为当前项目或工作树。使用终端图标或按 **Cmd+J** 打开它。

使用终端验证更改、运行脚本和执行 Git 操作，而无需离开应用。

常见任务包括：

- `git status`
- `git pull --rebase`
- `pnpm test` 或 `npm test`
- `pnpm run lint` 或类似的项目命令

如果您定期运行任务，可以在[本地环境](https://developers.openai.com/codex/app/local-environments)中定义一个 **action**，为您的 Codex 应用窗口顶部添加一个快捷按钮。

<Callout type="note">
**Cmd+K** 打开 Codex 应用中的命令面板。它不会清除终端。要清除终端，请使用 **Ctrl+L**。
</Callout>

### 语音听写

使用您的语音提示 Codex。在作曲家可见时按住 **Ctrl+M** 并开始说话。您的语音将被转录。编辑转录的提示词或发送以让 Codex 开始工作。

### 与 IDE 扩展同步

如果您在编辑器中安装了 [Codex IDE 扩展](https://developers.openai.com/codex/ide)，您的 Codex 应用和 IDE 扩展在同一个项目中时会自动同步。

当它们同步时，您会在 Codex 应用编辑器中看到一个 **IDE 上下文** 选项。启用"自动上下文"后，Codex 应用会跟踪您正在查看的文件，因此您可以间接引用它们（例如，"此文件是关于什么的？"）。您还可以在 Codex 应用中看到在 IDE 扩展中运行的线程，反之亦然。

如果您不确定应用是否包含上下文，请关闭它并再次提出相同的问题以比较结果。

### 批准和沙箱

您的批准和沙箱设置限制了 Codex 操作。

- 批准决定了 Codex 在运行命令之前何时暂停以获得许可。
- 沙箱控制了 Codex 可以使用哪些目录和网络访问。

当您看到"批准一次"或"在此会话中批准"等提示时，您是在授予不同范围的工具执行权限。如果不确定，请批准最窄的选项并继续迭代。

默认情况下，Codex 将工作范围到当前项目。在大多数情况下，这是正确的约束。

如果您的任务需要跨多个存储库或目录工作，请优先打开单独的项目或使用工作树，而不是要求 Codex 在项目根目录之外漫游。

有关 Codex 如何处理沙箱的详细信息，请查看[安全文档](https://developers.openai.com/codex/security)。

### MCP 支持

Codex 应用、CLI 和 IDE 扩展共享[模型上下文协议 (MCP)](https://developers.openai.com/codex/mcp) 设置。如果您已经在其中一个配置了 MCP 服务器，它们会自动被其他采用。要配置新服务器，请打开应用设置中的 MCP 部分，启用推荐的服务器或将新服务器添加到您的配置中。

### 网页搜索

Codex 附带了一个官方的网页搜索工具。对于 Codex IDE 扩展中的本地任务，Codex 默认启用网页搜索并从网页搜索缓存中提供结果。如果您将沙箱配置为[完全访问](https://developers.openai.com/codex/security)，网页搜索默认为实时结果。请参阅[基本配置](https://developers.openai.com/codex/config-basic)以禁用网页搜索或切换到获取最新数据的实时结果。

### 图像输入

您可以将图像拖放到提示词编辑器中以将它们作为上下文包含。在拖放图像时按住 **Shift** 以将图像添加到上下文。

您还可以要求 Codex 查看您系统上的图像。通过给 Codex 截取您正在工作的应用程序的截图工具，Codex 可以验证它正在做的工作。

### 通知

默认情况下，Codex 应用在任务完成或需要批准而应用在后台时发送通知。

在 Codex 应用设置中，您可以选择从不发送通知，或始终发送通知，即使应用处于焦点状态。

### 保持计算机唤醒

由于您的任务可能需要一段时间才能完成，您可以通过在应用设置中启用"运行时防止睡眠"开关来让 Codex 应用防止计算机进入睡眠状态。
