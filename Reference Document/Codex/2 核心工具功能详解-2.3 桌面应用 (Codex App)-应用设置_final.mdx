# Codex 应用设置

使用设置面板来调整 Codex 应用的行为方式、打开文件的方式以及如何连接工具。从应用菜单打开 [**Settings**](codex://settings) 或按 **Cmd**+**,**。

<AdPlaceholder />

## 常规

选择文件打开的位置以及线程中显示多少命令输出。您还可以要求 **Cmd**+**Enter** 用于多行提示词，或在线程运行时防止睡眠。

## 外观

选择主题，决定窗口是否 solid，并调整 UI 或代码字体。字体选择应用于整个应用，包括差异审查面板和终端。

## 通知

选择何时显示完成通知，以及应用是否应提示获取通知权限。

## 代理配置

应用中的 Codex 代理继承与 IDE 和 CLI 扩展相同的配置。使用应用内控件进行常见设置，或编辑 `config.toml` 以获取高级选项。请参阅 [Codex 安全性](https://developers.openai.com/codex/security) 和[基本配置](https://developers.openai.com/codex/config-basic)了解更多详情。

## Git

使用 Git 设置来标准化分支命名，并选择 Codex 是否使用强制推送。您还可以设置 Codex 用于生成提交消息和拉取请求描述的提示词。

## 集成和 MCP

通过 MCP（模型上下文协议）连接外部工具。启用推荐的服务器或添加您自己的服务器。如果服务器需要 OAuth，应用会启动身份验证流程。这些设置也适用于 Codex CLI 和 IDE 扩展，因为 MCP 配置位于 `config.toml` 中。请参阅[模型上下文协议文档](https://developers.openai.com/codex/mcp)了解详情。

## 个性化

选择 **Friendly**、**Pragantic** 或 **None** 作为您的默认人格。使用 **None** 禁用个性化指令。您可以随时更新此设置。

您还可以添加自己的自定义指令。编辑自定义指令会更新您 [AGENTS.md 中的个人指令](https://developers.openai.com/codex/guides/agents-md)。

## 存档的线程

**存档的线程** 部分列出带有日期和项目上下文的存档聊天。使用 **Unarchive** 恢复线程。
