# 审查

审查面板可帮助您了解 Codex 更改了什么、提供有针对性的反馈，并决定保留什么。

它仅适用于位于 Git 存储库中的项目。如果您的项目还不是 Git 存储库，审查面板会提示您创建一个。

<AdPlaceholder />

## 它显示的更改

审查面板反映您 Git 存储库的状态，而不仅仅是 Codex 编辑的内容。这意味着它将显示：

- Codex 所做的更改
- 您自己所做的更改
- 存储库中任何其他未提交的更改

默认情况下，审查面板专注于**未提交的更改**。您也可以将范围切换到：

- **所有分支更改**（与基础分支的差异）
- **上一轮更改**（仅最新的助手轮次）

在本地工作时，您还可以在**未暂存**和**已暂存**更改之间切换。

## 浏览审查面板

- 单击文件名通常会在您选择的编辑器中打开该文件。您可以在[设置](https://developers.openai.com/codex/settings)中选择默认编辑器。
- 单击文件名背景可展开或折叠差异。
- 在按住 **Cmd** 的同时单击单行将在您的选择编辑器中打开该行。
- 如果您对更改满意，可以[暂存更改或回滚不喜欢的更改](#暂存和回滚文件)。

## 内联评论以获取反馈

内联评论让您可以将反馈直接附加到差异中的特定行。这通常是引导 Codex 进行正确修复的最快方式。

要留下内联评论：

1. 打开审查面板。
2. 悬停在线条上想要评论的位置。
3. 单击出现的 **+** 按钮。
4. 撰写您的反馈并提交。
5. 完成所有反馈后，发送消息回线程。

因为评论固定在一条线上，Codex 通常可以比一般指令更精确地响应。

内联评论被视为审查指导。留下评论后，发送一条明确意图的后续消息，例如"解决内联评论并保持范围最小。"

## 代码审查结果

如果您使用 `/review` 运行代码审查，评论将直接显示在审查面板中。

## 暂存和回滚文件

审查面板包括 Git 操作，以便您在提交之前塑造差异。

您可以在多个级别暂存、取消暂存或回滚更改：

- **整个差异**：使用审查标头中的操作按钮（例如，"全部暂存"或"全部回滚"）
- **每个文件**：暂存、取消暂存或回滚单个文件
- **每个块**：暂存、取消暂存或回滚单个块

当您想要接受部分工作时使用暂存，当您想要丢弃时使用回滚。

### 部分暂存状态

Git 可以在同一个文件中表示已暂存和未暂存的更改。当发生这种情况时，看起来面板在已暂存和未暂济视图中"显示同一个文件两次"。这是正常的 Git 行为。
