# 本地环境

本地环境让您可以配置工作树的设置步骤以及项目的常用操作。

您通过 [Codex 应用设置](codex://settings) 窗格配置本地环境。您可以将生成的文件检查到项目的 Git 存储库中以与他人共享。

Codex 将此配置存储在项目根目录下的 `.codex` 文件夹中。如果您的仓库包含多个项目，请打开包含共享 `.codex` 文件夹的项目目录。

## 设置脚本

由于工作树在与本地任务不同的目录中运行，您的项目可能没有完全设置，可能缺少未检查到存储库的依赖项或文件。设置脚本在 Codex 创建新工作树时自动运行在新线程的开始处。

使用此脚本来运行配置环境所需的任何命令，例如安装依赖项或运行构建过程。

例如，对于 TypeScript 项目，您可能希望使用设置脚本安装依赖项并进行初始构建：

```bash
npm install
npm run build
```

如果您的设置是特定于平台的，请为 macOS、Windows 或 Linux 定义设置脚本以覆盖默认值。

## 动作

<section class="feature-grid">

<div>

使用动作来定义常见任务，如启动应用程序的开发服务器或运行测试套件。这些动作出现在 Codex 应用顶部栏中以便快速访问。动作将在应用的[集成终端](https://developers.openai.com/codex/app/features#integrated-terminal)中运行。

动作有助于避免输入常见操作，如触发项目的构建或启动开发服务器。对于一次性快速调试，您可以直接使用集成终端。

例如，对于 Node.js 项目，您可以创建一个包含以下脚本的"运行"动作：

```bash
npm start
```

如果动作的命令是特定于平台的，请为 macOS、Windows 和 Linux 定义特定于平台的脚本。

要识别您的动作，请选择与每个动作关联的图标。
