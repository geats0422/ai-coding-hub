# Claude Code 概览与安装指南

Claude Code 是 Anthropic 开发的 AI 编程工具，它驻留在您的终端中，帮助您比以往任何时候都更快地将想法转化为代码。这个强大的工具集成了自动完成繁琐任务、智能调试和无缝集成。

<AdPlaceholder />

## 前置条件

在开始使用 Claude Code 之前，请确保您具备以下条件：

- **操作系统**: macOS 13.0+、Ubuntu 20.04+/Debian 10+ 或 Windows 10+（支持 WSL、WSL 2 或 Git for Windows）
- **硬件**: 4 GB+ RAM
- **网络连接**: 稳定的互联网连接
- **Shell 环境**: Bash 或 Zsh（推荐）
- **账户**: [Claude 订阅](https://claude.com/pricing)（Pro、Max、Teams 或 Enterprise）或 [Claude Console](https://console.anthropic.com/) 用户

<Note>
Windows ARM64 不支持本地会话。
</Note>

## 快速开始：30 秒安装

### 步骤 1：选择安装方式

Claude Code 提供多种安装方法以适应不同的工作流程。

**选项 A：本机安装（推荐）**

**macOS、Linux、WSL:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**
```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**
```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

<ProTip>
本机安装会在后台自动更新到最新版本，确保您始终拥有最新的功能和安全修复。
</ProTip>

**选项 B：Homebrew 安装**

```bash
brew install --cask claude-code
```

<Note>
Homebrew 安装不会自动更新。您需要定期运行 `brew upgrade claude-code` 来获取最新功能和安全修复。
</Note>

**选项 C：WinGet 安装**

```powershell
winget install Anthropic.ClaudeCode
```

<Note>
WinGet 安装不会自动更新。您需要定期运行 `winget upgrade Anthropic.ClaudeCode` 来获取最新功能和安全修复。
</Note>

<ProTip>
对于大多数用户，本机安装是最简单且推荐的方式。
</ProTip>

### 步骤 2：验证安装

安装完成后，运行以下命令验证：

```bash
claude doctor
```

这将检查您的安装类型和版本，确保一切正常工作。

### 步骤 3：登录您的账户

首次使用时，系统会提示您登录。您可以使用以下任一账户类型：

- **Claude 订阅**（推荐）: 订阅 Claude 的 [Pro 或 Max 计划](https://claude.com/pricing)，在一个地方管理您的账户
- **Claude Console**: 通过 [Claude Console](https://console.anthropic.com/) 连接，需要活跃的账单

登录命令：

```bash
claude
# 首次使用时系统会提示您登录
```

<Note>
当使用 Claude Console 时，系统会自动为您的组织创建一个"Claude Code"工作区，用于集中管理使用情况跟踪。
</Note>

<Note>
登录后，您的凭证将被存储，您无需再次登录。
</Note>

### 步骤 4：启动您的第一个会话

安装并登录后，导航到您的项目目录并启动 Claude Code：

```bash
cd your-project
claude
```

<ProTip>
确保您的项目目录是 Git 仓库，这样 Claude Code 可以更好地理解您的代码结构。
</ProTip>

您将看到 Claude Code 的欢迎界面，其中包含您的会话信息、最近的对话和最新更新。

## Claude Code 能为您做什么

### 从描述构建功能

用纯英文告诉 Claude 您想构建什么，它将制定计划、编写代码并确保其正常工作。

### 调试和修复问题

描述一个错误或粘贴错误消息，Claude Code 将分析您的代码库、识别问题并实施修复。

### 导航任何代码库

询问有关您团队代码库的任何问题，Claude Code 都能提供深思熟虑的答案。

### 自动化繁琐任务

修复棘手的 lint 问题、解决合并冲突和编写发布说明。从您的开发机器上用一个命令完成所有这些工作。

### 为什么开发者喜欢 Claude Code

#### 在您的终端中工作

不是另一个聊天窗口。不是另一个 IDE。Claude Code 在您已经工作的地方与您相遇，使用您已经喜欢的工具。

#### 采取行动

Claude Code 可以直接编辑文件、运行命令和创建提交。需要更多？[MCP](/zh-CN/mcp) 让 Claude 读取您在 Google Drive 中的设计文档、更新您在 Jira 中的工单或使用您的自定义开发者工具。

#### Unix 哲学设计

Claude Code 是可组合和可脚本化的。`tail -f app.log | claude -p "Slack me if you see any anomalies"` *有效*。

#### 企业就绪

使用 Claude API，或在 AWS、GCP 上托管。企业级的安全性、[隐私](/zh-CN/data-usage) 和[合规性](https://trust.anthropic.com/) 是内置的。

## 常见使用场景

### 场景 1：快速原型开发

```bash
# 描述功能
claude -p "创建一个 React 表单组件，包含姓名、邮箱、密码字段。使用 Material UI 设计。"

# 让 Claude 生成完整组件代码
# 提供样式参考
```

**优势**：
- 快速迭代和预览
- Claude 会自动处理表单验证和错误处理
- 可以轻松修改 UI 样式

### 场景 2：代码审查和重构

```bash
# 让 Claude 分析代码质量
claude -p "审查 src/auth 模块的安全性最佳实践和潜在的改进"

# 生成重构建议
claude -p "将 src/utils 函数转换为 TypeScript 类型，添加 JSDoc 类型注释"
```

### 场景 3：多项目协调

```bash
# 在一个目录中同时为多个项目工作
cd /workspace
for repo in project1 project2 project3; do
  claude -p "为每个项目生成独立的 Docker Compose 配置"
done
```

## 专业提示

### 提示 1：高效使用命令行模式

Claude Code 最强大的一点是它支持 `-p` 标志用于一次性任务：

```bash
# 修复所有 lint 错误
claude -p "运行 'npm run lint -- --fix' 并接受所有建议的更改"

# 快速代码生成
claude -p "生成一个 Express REST API 端点，包含 CRUD 操作"
```

<ProTip>
使用 `-p` 模式时，Claude Code 会专注于完成任务而不会请求交互式批准。
</ProTip>

### 提示 2：合理拆分复杂任务

对于大型任务，将其拆分为多个小步骤：

```bash
# 第一步
claude -p "设置数据库架构和迁移脚本"

# 第二步
claude -p "实现用户认证功能"

# 第三步
claude -p "编写 API 文档"
```

### 提示 3：充分利用 Claude 的代码理解能力

Claude Code 能深度理解大型代码库，充分利用这一能力：

```bash
# 让 Claude 分析整个项目架构
claude -p "识别设计模式和潜在的技术债务"

# 让 Claude 生成架构文档
```

## 常见问题排查

### 问题 1：安装失败

如果安装脚本执行失败：

**检查项目：**
```bash
# 验证系统要求
claude doctor
```

**检查网络连接：**
```bash
# 测试网络连接
curl -I https://claude.ai/
```

**重新尝试安装：**
```bash
# 对于本机安装
curl -fsSL https://claude.ai/install.sh | bash
```

### 问题 2：权限错误

如果 Claude Code 拒绝某个操作：

- 检查文件权限：`ls -la ~/.claude`
- 查看 Claude Code 配置：`cat ~/.claude/settings.json`
- 运行 `claude config` 交互式检查权限设置

### 问题 3：登录问题

**如果登录失败：**

- 检查账户类型是否正确
- 验证网络连接是否正常
- 尝试使用 `/login` 命令重新登录

**如果使用 Claude Console:**
- 确保您有活跃的账单
- 导航到 [Claude Console](https://console.anthropic.com/) 检查账单状态

### 问题 4：性能缓慢

**如果响应速度慢：**

- 检查网络延迟
- 尝试切换模型（如果订阅支持多个模型）
- 简化对话历史：`/compact` 删除不必要的内容

### 问题 5：上下文限制

**如果遇到令牌限制错误：**

- 这是大型文档或长会话的常见问题
- 使用 `/compact` 压缩对话历史
-  将复杂任务拆分为多个会话

### 问题 6：工具调用失败

**如果 Bash 命令执行失败：**

- 检查错误消息并重试
- 验证命令语法是否正确
