# Claude Code 交互模式与快捷键完全指南

熟练掌握 Claude Code 的键盘快捷键和交互模式可以显著提升您的开发效率。本指南提供完整的快捷键参考，包括常规控制、文本编辑、Vim 模式和后台任务管理等功能。

<AdPlaceholder />

## 前置条件

在开始学习快捷键和交互模式之前，请确保您具备以下条件：

- **Claude Code**：已安装并可运行的 Claude Code
- **终端应用程序**：支持的终端（iTerm2、VS Code、Windows Terminal 等）
- **基本命令行知识**：熟悉终端基本操作
- **键盘快捷键意识**：了解常用的键盘操作概念

## 步骤 1：掌握常规控制快捷键

### 步骤 1.1：理解快捷键的上下文依赖性

键盘快捷键可能因平台和终端而异。按 `?` 查看您的特定环境中可用的快捷键列表。

<Note>
macOS 用户需要注意：Option/Alt 键快捷键（`Alt+B`、`Alt+F`、`Alt+Y`、`Alt+M`、`Alt+P`）需要在终端中将 Option 配置为 Meta 键。
</Note>

### 步骤 1.2：配置 macOS Meta 键

**对于 iTerm2：**
1. 打开设置 → 配置文件 → 键
2. 将左/右 Option 键设置为"Esc+"

**对于 Terminal.app：**
1. 打开设置 → 配置文件 → 键盘
2. 勾选"使用 Option 作为 Meta 键"

**对于 VS Code：**
1. 打开设置 → 配置文件 → 键
2. 将左/右 Option 键设置为"Esc+"

<ProTip>
正确配置 Meta 键后，您可以更高效地进行单词导航和文本编辑操作。
</ProTip>

### 步骤 1.3：学习核心控制快捷键

**会话控制：**
| 快捷键 | 描述 | 上下文 |
|--------|------|--------|
| `Ctrl+C` | 取消当前输入或生成 | 标准中断操作 |
| `Ctrl+D` | 退出 Claude Code 会话 | EOF 信号 |
| `Ctrl+L` | 清除终端屏幕 | 保留对话历史 |
| `Esc + Esc` | 回退代码/对话 | 恢复到之前的状态 |

**输出和导航：**
| 快捷键 | 描述 | 上下文 |
|--------|------|--------|
| `Ctrl+O` | 切换详细输出 | 显示详细的工具使用和执行情况 |
| `Ctrl+R` | 反向搜索命令历史 | 交互式搜索以前的命令 |
| `Ctrl+G` | 在默认文本编辑器中打开 | 编辑您的提示或自定义响应 |

**权限和模式切换：**
| 快捷键 | 描述 | 上下文 |
|--------|------|--------|
| `Shift+Tab` 或 `Alt+M` | 切换权限模式 | 在自动接受模式、Plan Mode 和正常模式之间切换 |
| `Option+P` 或 `Alt+P` | 切换模型 | 在不清除提示的情况下切换模型 |
| `Option+T` 或 `Alt+T` | 切换扩展思考 | 启用或禁用扩展思考模式 |

## 步骤 2：掌握文本编辑快捷键

### 步骤 2.1：剪切和粘贴操作

**行内编辑：**
| 快捷键 | 描述 | 操作效果 |
|--------|------|----------|
| `Ctrl+K` | 删除到行尾 | 存储已删除的文本以供粘贴 |
| `Ctrl+U` | 删除整行 | 存储已删除的文本以供粘贴 |
| `Ctrl+Y` | 粘贴已删除的文本 | 粘贴用 `Ctrl+K` 或 `Ctrl+U` 删除的文本 |

**粘贴历史循环：**
| 快捷键 | 描述 | 操作效果 |
|--------|------|----------|
| `Alt+Y`（在 `Ctrl+Y` 之后） | 循环粘贴历史 | 粘贴后，循环浏览以前删除的文本 |

<ProTip>
掌握剪切和粘贴历史可以大大提高重复编辑的效率。
</ProTip>

### 步骤 2.2：单词导航

**单词级光标移动：**
| 快捷键 | 描述 | 平台要求 |
|--------|------|----------|
| `Alt+B` | 将光标向后移动一个单词 | macOS 需要将 Option 设置为 Meta |
| `Alt+F` | 将光标向前移动一个单词 | macOS 需要将 Option 设置为 Meta |

### 步骤 2.3：方向键和历史导航

- **Up/Down arrows**：导航命令历史，回忆以前的输入
- **Left/Right arrows**：在对话框选项卡之间循环导航

## 步骤 4：掌握多行输入方法

### 步骤 4.1：换行符输入方式

| 方法 | 快捷键 | 支持终端 |
|------|--------|----------|
| 快速转义 | `\` + `Enter` | 所有终端 |
| macOS 默认 | `Option+Enter` | macOS |
| Shift+Enter | `Shift+Enter` | iTerm2、WezTerm、Ghostty、Kitty |
| 控制序列 | `Ctrl+J` | 所有终端 |
| 粘贴模式 | 直接粘贴 | 对于代码块、日志 |

### 步骤 4.2：配置 Shift+Enter

Shift+Enter 在以下终端中开箱即用：
- iTerm2
- WezTerm
- Ghostty
- Kitty

对于其他终端（VS Code、Alacritty、Zed、Warp），运行以下命令：

```bash
/terminal-setup
```

此命令会自动安装必要的键绑定。

## 步骤 7：使用命令历史

### 步骤 7.1：理解历史存储

Claude Code 为当前会话维护命令历史：
- 历史按工作目录存储
- 使用 `/clear` 命令清除历史
- 使用上/下箭头导航历史
- 历史扩展（`!`）默认禁用

### 步骤 7.2：使用 Ctrl+R 反向搜索

**搜索步骤：**

1. **开始搜索**：按 `Ctrl+R` 激活反向历史搜索
2. **键入查询**：输入文本以在以前的命令中搜索
3. **导航匹配**：再次按 `Ctrl+R` 循环浏览较旧的匹配
4. **接受匹配**：
   - 按 `Tab` 或 `Esc` 接受当前匹配并继续编辑
   - 按 `Enter` 接受并立即执行命令
5. **取消搜索**：
   - 按 `Ctrl+C` 取消并恢复原始输入
   - 在空搜索上按 `Backspace` 取消

## 步骤 8：使用后台任务

### 步骤 8.1：理解后台运行机制

Claude Code 支持在后台运行 bash 命令，允许您在长时间运行的进程执行时继续工作。

**后台运行方式：**
- 提示 Claude Code 在后台运行命令
- 按 `Ctrl+B` 将常规 Bash 工具调用移到后台

<Note>
Tmux 用户必须按两次 Ctrl+B，因为 tmux 使用它作为前缀键。
</Note>

### 步骤 8.2：后台任务功能

**主要功能：**
- 输出被缓冲，Claude 可以使用 TaskOutput 工具检索
- 后台任务具有唯一 ID 用于跟踪和输出检索
- 当 Claude Code 退出时，后台任务会自动清理

**常见的适合后台运行的命令：**
- 构建工具（webpack、vite、make）
- 包管理器（npm、yarn、pnpm）
- 测试运行器（jest、pytest）
- 开发服务器
- 长时间运行的进程（docker、terraform）

<ProTip>
使用后台任务可以在等待编译或测试时继续其他开发工作。
</ProTip>

### 步骤 8.3：使用 Bash 模式

通过在输入前加上 `!` 直接运行 bash 命令：

```bash
! npm test
! git status
! ls -la
```

**Bash 模式特点：**
- 将命令及其输出添加到对话上下文
- 显示实时进度和输出
- 支持后台运行长时间运行的命令
- 不需要 Claude 解释或批准命令
- 支持基于历史的自动完成

## 步骤 9：使用任务列表

### 步骤 9.1：理解任务列表功能

在处理复杂的多步工作时，Claude 创建任务列表来跟踪进度。

**任务列表操作：**
- 按 `Ctrl+T` 切换任务列表视图（显示一次最多 10 个任务）
- 直接询问 Claude 显示所有任务："show me all tasks"
- 直接询问 Claude 清除所有任务："clear all tasks"
- 任务在上下文压缩中持续存在

### 步骤 9.2：跨会话共享任务列表

要共享任务列表，请设置 `CLAUDE_CODE_TASK_LIST_ID`：

```bash
CLAUDE_CODE_TASK_LIST_ID=my-project claude
```

## 常见问题排查

### 问题 1：快捷键不工作

**解决方案：**
1. 按 `?` 检查当前环境可用的快捷键
2. 确认终端支持该快捷键
3. 检查是否有快捷键冲突
4. 重启终端应用程序

### 问题 2：Vim 模式命令失效

**解决方案：**
1. 确认已启用 Vim 模式（运行 `/vim`）
2. 检查是否处于正确的模式
3. 验证命令语法是否正确

### 问题 3：后台任务无法启动

**解决方案：**
1. 检查环境变量 `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS`
2. 确认命令语法正确
3. 验证后台任务是否被系统支持

### 问题 4：命令历史不显示

**解决方案：**
1. 检查是否使用了 `/clear` 清除了历史
2. 确认工作目录是否正确
3. 重启 Claude Code 会话

### 问题 5：权限模式切换无效

**解决方案：**
1. 确认快捷键是否正确（`Shift+Tab` 或 `Alt+M`）
2. 检查权限配置是否允许模式切换
3. 参考权限文档进行正确配置
