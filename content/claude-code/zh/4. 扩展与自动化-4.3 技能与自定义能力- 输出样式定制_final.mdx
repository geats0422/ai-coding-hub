# Claude Code 输出样式完全指南

输出样式是定制 Claude Code 响应方式的强大机制，让您可以将 Claude Code 应用于软件工程之外的多种场景。无论是教学辅导、数据分析还是文档写作，通过选择不同的输出样式，Claude 能够以最适合您需求的方式进行交互。理解并掌握输出样式的使用，将帮助您在各种工作场景中都能获得最佳的 AI 辅助体验。

## 前置条件

本指南面向希望定制 Claude Code 响应方式的开发者，建议您已具备以下条件：

- 熟悉 Claude Code 的基本使用方法
- 了解 YAML frontmatter 的基本语法
- 对系统提示的概念有基本理解

## 内置输出样式

Claude Code 提供了三种内置输出样式，每种都针对不同的使用场景进行了优化。

### 默认样式

默认输出样式是 Claude Code 的标准系统提示配置，专门针对高效的软件工程任务进行了优化。这个样式强调简洁的回复、快速的执行和实用的问题解决，是日常开发工作的最佳选择。

### Explanatory 样式

Explanatory 样式在帮助您完成软件工程任务的同时，会主动提供教育性的"Insights"。这个样式特别适合想要理解代码实现选择和代码库模式的开发者。如果您想深入了解"为什么这样设计"而不仅仅是"怎么做"，这个样式将是理想选择。

### Learning 样式

Learning 样式采用协作式的边学边做模式。Claude 不仅会在编码时分享 Insights，还会要求您主动贡献小的、战略性代码片段。在这个模式下，Claude 会在您的代码中添加 `TODO(human)` 标记，指明需要您自己实现的部分，真正实现教学相长。

## 输出样式的工作原理

### 系统提示修改机制

输出样式通过直接修改 Claude Code 的系统提示来实现其效果。了解这个机制有助于您更好地理解和使用自定义输出样式。

所有输出样式都会进行以下操作。首先，它们会排除高效输出的通用说明，比如简洁回复的要求。其次，自定义输出样式默认会排除编码相关的指令，除非您将 `keep-coding-instructions` 设置为 true。第三，所有输出样式都会在系统提示的末尾追加自己的自定义说明内容。第四，整个对话期间会定期触发提醒，确保 Claude 遵守输出样式的特殊说明。

### 更改输出样式

有两种方式可以切换输出样式。使用 `/output-style` 命令访问交互式菜单进行选择，这个命令也可以从 `/config` 菜单中调用。或者直接使用 `/output-style [style]` 命令快速切换，例如 `/output-style explanatory` 会直接切换到 Explanatory 样式。

这些更改会保存在项目级别的 `.claude/settings.local.json` 文件中，您也可以直接编辑不同级别的设置文件中的 `outputStyle` 字段来配置。

## 创建自定义输出样式

### 基本结构

自定义输出样式是包含 frontmatter 和正文的 Markdown 文件，frontmatter 定义元数据，正文包含要添加到系统提示的指令：

```markdown
---
name: My Custom Style
description:
  A brief description of what this style does
---

# Custom Style Instructions

You are an interactive CLI tool that helps users with software engineering
tasks. [Your custom instructions here...]

## Specific Behaviors

[Define how the assistant should behave in this style...]
```

### 存储位置

您可以在两个级别保存自定义输出样式。用户级样式存放在 `~/.claude/output-styles/` 目录下，对所有项目可见。项目级样式存放在 `.claude/output-styles/` 目录下，仅对当前项目有效。

### Frontmatter 配置

输出样式支持以下 frontmatter 字段：

| 字段 | 用途 | 默认值 |
|------|------|--------|
| `name` | 输出样式名称 | 从文件名继承 |
| `description` | 样式描述，仅在 UI 中显示 | 无 |
| `keep-coding-instructions` | 是否保留编码相关提示 | false |

`keep-coding-instructions` 字段特别有用，如果您希望在自定义样式中仍然保留 Claude Code 的核心编码能力，可以将其设置为 true。

## 与相似功能的对比

### 输出样式 vs CLAUDE.md

这是一个重要的区分。输出样式会完全替换 Claude Code 默认系统提示中与特定用途相关的部分。而 CLAUDE.md 和 `--append-system-prompt` 都不会编辑默认系统提示。CLAUDE.md 将内容作为用户消息追加到系统提示之后，`--append-system-prompt` 将内容附加到系统提示末尾。简单来说，输出样式是"替换"，CLAUDE.md 和 `--append-system-prompt` 是"追加"。

### 输出样式 vs 子代理

输出样式直接影响主代理循环，仅修改系统提示。子代理则用于处理特定任务，可以包含额外的配置，如要使用的模型、可用的工具集以及何时调用代理的上下文信息。子代理更适合需要专门处理的复杂任务，输出样式更适合全局性的行为调整。

### 输出样式 vs 技能

输出样式修改的是 Claude 的响应方式，包括格式、语气和结构，一旦选择就会始终保持激活状态。技能则是特定于任务的提示，可以使用 `/skill-name` 调用，或在 Claude 判断相关时自动加载。使用输出样式来实现一致的格式化偏好，使用技能来实现可重用的工作流程和任务自动化。

## 实际应用场景

### 教学辅导场景

结合 Learning 样式和自定义的教学提示，Claude 可以成为优秀的编程导师。它会在关键知识点提供解释，并留出空间让您自己思考和实现，培养独立解决问题的能力。

### 文档写作场景

创建一个专注于清晰表达的输出样式，让 Claude 以更加详尽、结构化的方式生成文档，适合技术文档撰写、教程编写等工作。

### 代码审查场景

定制输出样式让 Claude 在代码审查时提供更加详细和建设性的反馈，包括设计决策分析、性能考虑和安全建议。

## 总结

输出样式为 Claude Code 提供了灵活的定制能力，让同一个工具能够适应不同的工作场景。从选择内置样式开始，了解其工作原理后尝试创建自定义样式，将帮助您最大化 Claude Code 的价值。记住，输出样式是全局行为调整，技能是任务级自动化，两者配合使用效果最佳。
