# Claude Code 插件市场完全指南

插件市场是扩展 Claude Code 能力的核心渠道，通过安装插件，您可以快速获得新的命令、代理、钩子和 MCP 服务器集成。官方市场预装了多种常用插件，第三方市场则提供了更多专业化的扩展方案。掌握插件市场的使用方法，将帮助您根据项目需求灵活定制开发环境，大幅提升工作效率。

<AdPlaceholder />

## 前置条件

在开始使用插件市场之前，请确保满足以下条件：

- 已安装 Claude Code 1.0.33 或更高版本
- 对命令行操作有一定经验
- 了解基本的 Git 使用方法
- 具备网络连接以访问插件仓库

## 插件市场工作原理

插件市场采用目录式的插件分发模式，使用过程分为两个步骤。首先添加市场，这只是注册市场目录并不会安装任何插件。其次浏览目录并安装您需要的特定插件。这种模式类似于添加应用商店——添加商店让您可以访问浏览其集合，但您仍然单独选择要下载的应用。

官方 Anthropic 市场（`claude-plugins-official`）在启动 Claude Code 时自动可用，无需手动添加。运行 `/plugin` 并转到**发现**选项卡即可浏览可用内容。

## 官方市场插件类型

### 代码智能插件

代码智能插件帮助 Claude 更深入地理解您的代码库，提供跳转定义、查找引用、即时类型错误检查等功能。这些插件基于语言服务器协议（LSP）实现，与 VS Code 的代码智能使用相同的技术架构。

| 语言 | 插件 | 所需二进制文件 |
|------|------|---------------|
| C/C++ | clangd-lsp | clangd |
| C# | csharp-lsp | csharp-ls |
| Go | gopls-lsp | gopls |
| Java | jdtls-lsp | jdtls |
| Lua | lua-lsp | lua-language-server |
| PHP | php-lsp | intellectense |
| Python | pyright-lsp | pyright-langserver |
| Rust | rust-analyzer-lsp | rust-analyzer |
| Swift | swift-lsp | sourcekit-lsp |
| TypeScript | typescript-lsp | typescript-language-server |

如果您已经安装了语言服务器，Claude 可能会在打开项目时提示安装相应插件。

### 外部集成插件

这些插件预配置了 MCP 服务器，实现与外部服务的无缝连接：

**源代码控制**：`github`、`gitlab` 提供版本控制集成。**项目管理**：`atlassian` 连接 Jira 和 Confluence，`asana`、`linear`、`notion` 对接主流项目管理工具。**设计工具**：`figma` 实现设计资源访问。**基础设施**：`vercel`、`firebase`、`supabase` 提供云服务集成。**通信**：`slack` 实现团队协作通知。**监控**：`sentry` 集成错误追踪。

### 开发工作流插件

为常见开发任务添加专用命令和代理：

`commit-commands` 提供完整的 Git 提交工作流，包括提交、推送和 PR 创建功能。`pr-review-toolkit` 是专门用于审查拉取请求的代理工具。`agent-sdk-dev` 帮助使用 Claude Agent SDK 构建自定义工具。`plugin-dev` 提供了创建插件的完整工具包。

### 输出样式插件

自定义 Claude 的响应方式：`explanatory-output-style` 提供关于实现选择的教育见解，`learning-output-style` 用于技能构建的交互式学习模式。

## 快速上手：添加演示市场

Anthropic 还维护一个演示插件市场（`anthropics/claude-code`），包含展示插件系统可能性的示例插件。与官方市场不同，需要手动添加。

### 步骤 1：添加市场

在 Claude Code 中运行以下命令添加演示市场：

```shell
/plugin marketplace add anthropics/claude-code
```

这会下载市场目录并使其插件对您可用。

### 步骤 2：浏览可用插件

运行 `/plugin` 打开插件管理器。这是一个选项卡式界面，包含四个选项卡：

**发现**：从所有市场浏览可用插件。**已安装**：查看和管理已安装的插件。**市场**：添加、删除或更新添加的市场。**错误**：查看任何插件加载错误。使用 Tab 键循环切换选项卡（Shift+Tab 向后切换）。

### 步骤 3：安装插件

选择插件查看详细信息，然后选择安装范围：

**用户范围**（默认）：在所有项目中为自己安装。**项目范围**：为此仓库上的所有协作者安装（添加到 `.claude/settings.json`）。**本地范围**：仅在此仓库中为自己安装，不与协作者共享。

从命令行安装：

```shell
/plugin install commit-commands@anthropics-claude-code
```

### 步骤 4：使用新插件

安装后插件命令立即可用。插件命令由插件名称命名空间，例如 `commit-commands` 提供 `/commit-commands:commit` 等命令。

对文件进行更改并运行：

```shell
/commit-commands:commit
```

这会暂存更改、生成提交消息并创建提交。

## 添加市场

### 从 GitHub 添加

使用 `owner/repo` 格式添加包含 `.claude-plugin/marketplace.json` 文件的 GitHub 仓库：

```shell
/plugin marketplace add anthropics/claude-code
```

### 从其他 Git 主机添加

通过完整 URL 添加任何 Git 仓库，适用于 GitLab、Bitbucket 和自托管服务器：

使用 HTTPS：
```shell
/plugin marketplace add https://gitlab.com/company/plugins.git
```

使用 SSH：
```shell
/plugin marketplace add git@gitlab.com:company/plugins.git
```

添加特定分支或标签：
```shell
/plugin marketplace add https://gitlab.com/company/plugins.git#v1.0.0
```

### 从本地路径添加

添加包含 `.claude-plugin/marketplace.json` 文件的本地目录：

```shell
/plugin marketplace add ./my-marketplace
```

或添加 `marketplace.json` 文件的直接路径：
```shell
/plugin marketplace add ./path/to/marketplace.json
```

### 从远程 URL 添加

通过 URL 添加远程 `marketplace.json` 文件：

```shell
/plugin marketplace add https://example.com/marketplace.json
```

## 安装插件

### 基本安装

添加市场后直接安装插件：

```shell
/plugin install plugin-name@marketplace-name
```

### 选择安装范围

通过交互式界面选择安装范围：运行 `/plugin`，转到**发现**选项卡，在插件上按 Enter，选择所需范围。

**用户范围**（默认）：在所有项目中为自己安装。**项目范围**：为此仓库上的所有协作者安装。**本地范围**：仅在此仓库中为自己安装。

### 卸载命令

```shell
/plugin uninstall plugin-name@marketplace-name
```

使用 `--scope` 选项针对特定范围：
```shell
claude plugin install formatter@your-org --scope project
claude plugin uninstall formatter@your-org --scope project
```

## 管理已安装插件

### 使用交互式界面

运行 `/plugin` 转到**已安装**选项卡，可以查看、启用、禁用或卸载插件。

### 使用命令行

禁用插件而不卸载：
```shell
/plugin disable plugin-name@marketplace-name
```

重新启用已禁用的插件：
```shell
/plugin enable plugin-name@marketplace-name
```

### 列出所有插件

```shell
/plugin marketplace list
```

### 刷新市场

```shell
/plugin marketplace update marketplace-name
```

## 自动更新配置

### 启用自动更新

为市场启用自动更新后，Claude Code 会在启动时刷新市场数据并将已安装插件更新到最新版本。

通过 UI 为单个市场切换自动更新：运行 `/plugin` 打开插件管理器，选择**市场**，从列表中选择市场，选择**启用自动更新**或**禁用自动更新**。

### 管理策略

官方 Anthropic 市场默认启用自动更新。第三方和本地开发市场默认禁用自动更新。

完全禁用所有自动更新：
```shell
export DISABLE_AUTOUPDATER=true
```

禁用 Claude Code 更新但保持插件更新：
```shell
export DISABLE_AUTOUPDATER=true
export FORCE_AUTOUPDATE_PLUGINS=true
```

### 团队市场配置

团队管理员可以通过将市场配置添加到 `.claude/settings.json` 来为项目设置自动市场安装。当团队成员克隆仓库时，Claude Code 会提示他们安装这些市场和插件。

## 故障排除

### 命令无法识别

如果 `/plugin` 命令未出现：

1. **检查版本**：运行 `claude --version`，插件需要 1.0.33 或更高版本
2. **更新 Claude Code**：
   - Homebrew: `brew upgrade claude-code`
   - npm: `npm update -g @anthropic-ai/claude-code`
   - 本地安装程序：从设置页面重新运行安装命令
3. **重启 Claude Code**：更新后重启终端

### 常见问题

**市场未加载**：验证 URL 是否可访问以及 `.claude-plugin/marketplace.json` 是否存在于该路径。

**插件安装失败**：检查插件源 URL 是否可访问以及仓库是否为公开访问。

**安装后文件未找到**：插件被复制到缓存，引用插件目录外文件的路径将不起作用。

**插件技能未出现**：使用 `rm -rf ~/.claude/plugins/cache` 清除缓存，重启 Claude Code，然后重新安装插件。

## 安全注意事项

在安装插件之前，请确保您信任该插件。Anthropic 不控制插件中包含的 MCP 服务器、文件或其他软件，也无法验证它们是否按预期工作。建议检查每个插件的主页了解其功能和数据访问权限，特别是涉及外部服务集成的插件。

## 总结

插件市场为 Claude Code 提供了强大的扩展能力。从官方市场的代码智能和外部集成插件开始，逐步探索第三方市场提供的专业工具。合理管理安装范围和自动更新策略，可以帮助您在团队协作中保持一致的插件配置。遇到问题时，善用故障排除指南和社区资源进行解决。
