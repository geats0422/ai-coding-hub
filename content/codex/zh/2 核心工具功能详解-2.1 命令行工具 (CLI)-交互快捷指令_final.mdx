# Codex CLI 斜线命令

斜杠命令让您可以通过键盘快速控制Codex。在编辑器中输入 `/` 以打开斜杠弹出窗口，选择命令，Codex 将执行诸如切换模型、调整权限或总结长对话之类的操作，而无需离开终端。

本指南向您展示如何：

- 找到适合任务的正确内置斜线命令
- 使用 `/model`、`/personality`、`/permissions` 和 `/status` 等命令启动活动会话

<AdPlaceholder />

## 内置斜杠命令

Codex 附带以下命令。打开斜杠弹出窗口并开始键入命令名称以过滤列表。

| 命令 | 目的 | 何时使用 |
| ---- | ---- | -------- |
| `/permissions` |设置Codex无需事先询问即可执行的操作。 |放宽或收紧会话内的批准要求，例如在自动和只读之间切换。 |
| `/apps` |浏览应用程序（连接器）并将其插入到您的提示词中。 |在要求 Codex 使用该应用程序之前，请将该应用程序附加为 `$app-slug`。 |
| `/compact` |总结可见对话以释放令牌。 |在长时间运行后使用，以便 Codex 保留关键点而不会耗尽上下文窗口。 |
| `/diff` |显示 Git 差异，包括 Git 尚未跟踪的文件。 |在提交或运行测试之前查看法典编辑。 |
| `/exit` |退出 CLI（与`/quit` 相同）。 |替代拼写；这两个命令都退出会话。 |
| `/feedback` |将日志发送给Codex维护者。 |报告问题或与支持人员共享诊断信息。 |
| `/init` |在当前目录生成`AGENTS.md`脚手架。 |捕获您正在工作的存储库或子目录的持久指令。 |
| `/logout` |从Codex退出。 |清除共享计算机上的本地凭据。 |
| `/mcp` |列出已配置的模型上下文协议 (MCP) 工具。 |检查Codex 在会话期间可以调用哪些外部工具。 |
| `/mention` |将文件附加到对话中。 |指向您想要 Codex 接下来检查的特定文件或文件夹。 |
| `/model` |选择活动模型（以及推理工作（如果可用））。 |在运行任务之前，在通用模型（例如`gpt-4.1-mini`）和更深层次的推理模型之间切换。 |
| `/plan` |切换到计划模式并可选择发送提示词。 |要求Codex在实施工作开始之前提出执行计划。 |
| `/personality` |选择响应的沟通方式。 |在不改变您的指令的情况下，使Codex 更简洁、更具解释性或更具协作性。 |
| `/ps` |显示实验后端终端及其最新输出。 |检查长时间运行的命令而不留下主记录。 |
| `/fork` |将当前对话分叉到一个新线程中。 |分支活动会话以探索新方法，而不会丢失当前记录。 |
| `/resume` |从对话列表中恢复已保存的对话。 |继续从之前的 CLI 会话开始工作，无需重新开始。 |
| `/new` |在同一 CLI 会话中开始新对话。 |当您想要在同一代码库中获取新的提示词而不离开 CLI 时，请重置聊天上下文。 |
| `/quit` |退出 CLI。 |立即离开会议。 |
| `/review` |请Codex 检查您的工作树。 |在 Codex 完成其工作后或当您需要第二双眼睛来检查本地更改时运行。 |
| `/status` |显示会话配置和令牌使用情况。 |确认活动模型、审批策略、可写根目录和剩余上下文容量。 |
| `/debug-config` |打印配置层和需求诊断。 |调试优先级和策略要求，包括实验网络限制。 |
| `/statusline` |交互式配置页脚字段。 |选择并重新排序页脚项目 (models/context/restrictions/git/tokens/sessions) 并保留到 config.toml。 |

<Callout type="note">
`/quit` 和 `/exit` 都将退出 CLI。仅在保存或提交任何重要工作后才使用它们。
</Callout>

<Callout type="note">
`/approvals` 命令仍然可以用作别名，但它不再出现在斜杠弹出列表中。
</Callout>

## 使用斜杠命令来控制你的会话

以下工作流程可以让您的会话保持在正轨上，而无需重新启动Codex。

### 使用`/model` 设置活动模型

1. 启动Codex并打开编辑器。
2. 输入`/model` 并按 Enter。
3. 在弹出的窗口中选择型号，例如`gpt-4.1-mini`或`gpt-4.1`。

预期结果：Codex 确认记录中的新模型。运行 `/status` 来验证更改。

### 使用`/personality`设置通讯方式

使用`/personality` 更改Codex 的通信方式，而无需重写您的提示文字。1. 在活动对话中，输入 `/personality` 并按 Enter。
2. 从弹出窗口中选择一种样式。

预期结果：Codex 确认记录中的新样式并在会话中的后续响应中使用它。

Codex 支持`friendly`、`pragmatic` 和`none` 个性。使用`none` 禁用个性化指令。

如果当前型号不支持个性化指令，Codex将隐藏此命令。

### 使用`/plan`切换到计划模式1. 输入 `/plan` 并按 Enter 将活动对话切换到计划模式。
2. 可选：提供内嵌提示文字（例如`/plan Propose a migration plan for this service`）。
3. 使用内联`/plan` 参数时，您可以粘贴内容或附加图像。预期结果：Codex 进入计划模式，并将可选的内联提示词作为第一个计划请求。

当任务已经运行时，`/plan` 暂时不可用。

### 使用`/permissions` 更新权限

1. 输入 `/permissions` 并按 Enter 键。
2. 选择符合您舒适度的认可预设，例如用于无人值守运行的`Auto`，或用于审阅编辑的`Read Only`。预期结果：Codex 宣布更新战略。以后的操作将遵循新的审批模式，直到您再次更改为止。

### 使用`/status`检查会话

1. 在任何对话中，键入`/status`。
2. 查看输出以了解活动模型、审批策略、可写根目录和当前令牌使用情况。

预期结果：您将看到类似于 `Codex status` 在 shell 中打印的摘要，确认 Codex 正在您期望的位置运行。

### 使用`/debug-config`检查配置层

1. 输入`/debug-config`。
2. 检查输出的配置层顺序（从最低优先级开始）、开/关状态和策略来源。预期结果：Codex 打印层诊断以及策略详细信息，例如 `allowed_approval_policies`、`allowed_sandbox_modes`、`mcp_servers`、`rules`、`enforce_residency` 和 `experimental_network`（在配置时）。

使用此输出来调试有效设置与`config.toml` 不同的原因。

### 使用`/statusline`配置页脚项

1. 输入`/statusline`。
2. 使用选择器切换和重新排序项目，然后确认。

预期结果：页脚状态行立即更新并保留到`config.toml`中的`tui.status_line`。可用的状态行项目包括模型、模型+推理、上下文统计、速率限制、git分支、令牌计数器、会话ID、当前目录/项目根目录和Codex版本。

### 使用`/ps`查看后台终端1. 输入`/ps`。
2. 查看后台终端列表及其状态。

预期结果：Codex 显示每个后台终端的命令以及最多三个最新的非空输出行，以便您可以快速查看进度。

使用`unified_exec`时会出现后台终端；否则列表可能为空。

### 使用`/compact` 简化记录

1. 经过长时间的交流后，输入`/compact`。
2. 确认 Codex 何时提供迄今为止的对话摘要。

预期结果：Codex 用简洁的摘要取代了前几轮，在保留关键细节的同时释放了上下文。

### 使用`/diff` 来查看更改

1. 输入`/diff`来检查Git diff。
2. 滚动 CLI 中的输出以查看编辑和添加的文件。预期内容：Codex 显示已暂存的更改、尚未暂存的更改以及 Git 尚未开始跟踪的文件，以便您可以决定保留哪些内容。

### 使用`/mention` 突出显示文件

1. 键入`/mention`，后跟路径，例如`/mention src/lib/api.ts`。
2. 在弹出的窗口中选择匹配结果。

预期结果：Codex 将文件添加到对话中，确保后续回合直接引用它。

### 使用`/new` 开始新对话

1. 输入 `/new` 并按 Enter 键。

预期结果：Codex 在同一 CLI 会话中启动新对话，因此您无需离开终端即可切换任务。

### 使用`/resume` 恢复已保存的对话1. 输入 `/resume` 并按 Enter 键。
2. 从已保存的会话选择器中选择所需的会话。预期结果：Codex 重新加载所选对话的记录，以便您可以从上次中断的地方继续，同时保持原始历史记录不变。

### 使用`/fork` 分叉当前对话

1. 输入 `/fork` 并按 Enter 键。

预期结果：Codex 将当前对话克隆到具有新 ID 的新线程中，保持原始记录不变，以便您可以并行探索替代方案。

如果您想分叉已保存的会话而不是当前会话，请运行 `Codex fork` 以在终端中打开会话选择器。

### 使用`/init`生成`AGENTS.md`1. 在您希望Codex 查找持久指令的目录中运行`/init`。
2. 查看生成的`AGENTS.md` 并根据您的存储库约定对其进行编辑。

预期结果：Codex 创建一个 `AGENTS.md` 支架，您可以对其进行优化并提交以供将来的会话使用。

### 使用`/review` 请求工作树审查

1. 输入`/review`。
2. 如果您想检查确切的文件更改，请跟进`/diff`。

预期结果：Codex 总结了它在工作树中发现的问题，重点关注行为变化和缺失测试。除非您在`config.toml` 中设置`review_model`，否则它将使用当前会话模型。

### 使用`/mcp`列出MCP工具

1. 输入`/mcp`。
2. 检查列表以确认哪些 MCP 服务器和工具可用。预期结果：您可以看到 Codex 可以在此会话中调用的已配置模型上下文协议 (MCP) 工具。

### 使用`/apps` 浏览应用程序

1. 输入`/apps`。
2. 从列表中选择一个应用程序。

预期结果：Codex 将应用提及作为 `$app-slug` 插入编辑器，以便您可以立即要求 Codex 使用它。

### 使用`/feedback` 发送反馈1。输入 `/feedback` 并按 Enter 键。
2. 按照提示添加日志或诊断信息。

预期结果：Codex收集请求的诊断信息并将其提交给维护者。

### 使用`/logout`退出

1. 输入 `/logout` 并按 Enter。

预期结果：Codex 清除当前用户会话的本地凭据。

### 使用`/quit` 或`/exit` 退出CLI1. 输入`/quit`（或`/exit`）并按 Enter。

预期结果：Codex 立即退出。首先保存或提交任何重要的工作。
