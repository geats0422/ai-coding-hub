# Codex 应用程序

<AdPlaceholder />

Codex 应用程序是一种以桌面为中心的体验，用于并行处理 Codex 线程，具有内置工作树支持、自动化和 Git 功能。

Codex 包含在 ChatGPT Plus、Pro、Business、Edu 和 Enterprise 计划中。了解[包含的内容](https://developers.OpenAI.com/Codex/pricing)。

*图像描述：Codex应用程序窗口显示项目侧边栏、活动线程和审阅面板的界面布局。 *

## 开始使用

Codex 应用程序目前可在 macOS (Apple Silicon) 上使用。

<Callout type="step">
1.下载并安装Codex应用程序

    Codex 应用程序目前仅在 macOS 上可用。

    
<Callout type="download">
    下载macOS版本
    </Callout>
<Callout type="note">
    注册 Windows 和 Linux 通知
    </Callout>

2.打开Codex并登录

    下载并安装Codex应用程序后，打开它并使用您的ChatGPT帐户或OpenAI API密钥登录。如果您使用OpenAI API登录，某些功能（例如[云线程](https://developers.OpenAI.com/Codex/prompting#threads)）可能无法使用。

3. 选择项目

    选择您希望 Codex 工作的项目文件夹。

    如果您以前使用过 Codex 应用程序、CLI 或 IDE 扩展，您将看到您过去处理过的项目。

4. 发送您的第一条消息

    选择您的项目后，请确保选中**本地**以使Codex在您的计算机上运行，然后将您的第一条消息发送到Codex。您可以向Codex询问有关该项目或您的计算机的任何问题。以下是一些示例：

    - “告诉我这个项目”
    - “在此存储库中构建经典的贪吃蛇游戏。”
    - “通过最少的、高可信度的更改来查找并修复我的代码库中的错误。”

    如果您需要更多灵感，请查看[探索部分](https://developers.OpenAI.com/Codex/explore)。
</Callout>

## 使用Codex 应用程序

### 跨项目多任务处理

使用 Codex 应用程序窗口跨项目运行任务。为每个代码库添加一个项目并根据需要切换它们。If you've used [Codex CLI](https://developers.OpenAI.com/Codex/cli), the project is like starting a session in a specific directory.If you are working in a single repository with two or more applications or packages, split the different projects into separate app projects so that only the files for that project are included.

### 内置 Git 工具

Codex 应用程序在应用程序内提供常见的 Git 功能。

差异面板显示本地项目或工作树签出中更改的 Git 差异。您还可以向 Codex 添加内联注释，并暂存或回滚特定块或整个文件。

您还可以直接从 Codex 应用程序提交、推送和创建本地任务和工作树任务的拉取请求。

对于更高级的 Git 任务，请使用[集成终端](#集成终端)。

### 工作树支持创建新线程时，选择**本地**或**工作树**。 **本地**直接在您的项目中工作。 **工作树** 创建一个新的[Git 工作树](https://git-scm.com/docs/git-worktree)，以便更改与常规项目保持隔离。

当您想要尝试新想法而不触及当前工作时，或者当您希望 Codex 在同一项目中并行运行独立任务时，请使用 Worktree。

自动化在 Git 存储库的专用后台工作树中运行，并直接在非版本控制项目的工作目录中运行。

[详细了解如何在 Codex 应用中使用工作树。](https://developers.OpenAI.com/Codex/app/worktrees)

### 综合终端

每个线程都包含一个内置终端，范围仅限于当前项目或工作树。使用终端图标或按 **Cmd+J** 打开它。使用终端验证更改、运行脚本并执行 Git 操作，而无需离开应用程序。

常见任务包括：- `git status`
-`git pull --rebase`
- `pnpm test` 或 `npm test`
- `pnpm run lint` 或类似的项目命令

如果您定期运行任务，则可以在[本地环境](https://developers.OpenAI.com/Codex/app/local-environments) 中定义**操作**，以将快捷按钮添加到Codex 应用程序窗口的顶部。

<Callout type="note">
**Cmd+K** 打开 Codex 应用程序中的命令选项板。它不会清除终端。要清除终端，请使用 **Ctrl+L**。
</Callout>

### 语音听写使用语音提示Codex。当作曲家可见时按住 **Ctrl+M** 并开始讲话。您的讲话将被转录。编辑转录的提示文字或发送让Codex开始工作。

### 与 IDE 扩展同步

如果您的编辑器中安装了 [Codex IDE 扩展](https://developers.OpenAI.com/Codex/ide)，则您的 Codex 应用程序和 IDE 扩展在同一项目中时将自动同步。

当它们同步时，您将在 Codex 应用程序编辑器中看到 **IDE Con​​text** 选项。启用自动上下文后，Codex 应用程序会跟踪您正在查看的文件，以便您可以间接引用它们（例如，“这个文件是关于什么的？”）。您还可以在 Codex 应用程序的 IDE 扩展中看到运行的线程，反之亦然。如果您不确定应用程序是否包含上下文，请将其关闭并再次询问相同的问题以比较结果。

### 批准和沙箱

您的批准和沙箱设置限制Codex操作。

- 批准确定 Codex 在运行命令之前何时暂停以获得权限。
- 沙箱控制Codex可以使用哪些目录和网络访问。当您看到“批准一次”或“在此会话中批准”等提示时，您正在向该工具授予不同范围的执行权限。如果您不确定，请批准最窄的选项并继续迭代。

默认情况下，Codex 将在当前项目范围内工作。在大多数情况下，这是正确的约束。如果您的任务需要跨多个存储库或目录进行工作，则更愿意打开单独的项目或使用工作树，而不是要求 Codex 在项目根目录之外漫游。

有关Codex 如何处理沙箱的更多信息，请查看[安全文档](https://developers.OpenAI.com/Codex/security)。

### MCP 支持

Codex 应用程序、CLI 和 IDE 扩展共享 [模型上下文协议 (MCP)](https://developers.OpenAI.com/Codex/mcp) 设置。如果您在其中一个中配置了 MCP 服务器，则它们将自动被另一个中采用。要配置新服务器，请打开应用程序设置中的 MCP 部分并启用推荐的服务器或将新服务器添加到您的配置中。

### 网络搜索Codex 附带官方网络搜索工具。对于 Codex IDE 扩展中的本地任务，Codex 默认启用 Web 搜索并提供来自 Web 搜索缓存的结果。如果您将沙箱配置为[完全访问](https://developers.OpenAI.com/Codex/security)，网络搜索默认为实时结果。请参阅[基本配置](https://developers.OpenAI.com/Codex/config-basic)禁用网络搜索或切换到实时结果以获取最新数据。

### 图像输入

您可以将图像拖放到提示词编辑器中，将它们作为上下文包含在内。拖放图像时按住 **Shift** 将图像添加到上下文中。您还可以要求Codex 在您的系统上查看图像。通过向 Codex 提供您正在处理的应用程序的屏幕截图，Codex 可以验证它是否运行良好。### 通知

默认情况下，Codex 应用程序在任务完成或需要批准且应用程序处于后台时发送通知。

在Codex应用程序设置中，您可以选择从不发送通知，或者即使在应用程序处于焦点状态时也始终发送通知。

### 保持计算机处于唤醒状态

由于您的任务可能需要一段时间才能完成，因此您可以通过在应用程序设置中启用“运行时防止睡眠”开关，让 Codex 应用程序阻止您的计算机进入睡眠状态。
