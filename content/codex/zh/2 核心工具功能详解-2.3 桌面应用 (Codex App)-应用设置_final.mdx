#Codex应用程序设置

使用设置面板调整Codex应用程序的行为方式、打开文件的方式以及连接工具的方式。从应用程序菜单中打开 [**Settings**](Codex://settings) 或按 **Cmd**+**,**。

<AdPlaceholder />

## 一般

选择文件打开位置以及线程中显示的命令输出量。您还可以要求 **Cmd**+**Enter** 来获取多行提示词，或者在线程运行时防止休眠。

## 外观

选择主题，决定窗口是否为实心，并调整 UI 或代码字体。字体选择适用于整个应用程序，包括差异审查面板和终端。

## 通知

选择何时显示完成通知以及应用程序是否应提示通知权限。

## 代理配置应用程序中的 Codex 代理继承与 IDE 和 CLI 扩展相同的配置。使用应用程序内控件进行常用设置，或编辑 `config.toml` 进行高级选项。请参阅[Codex 安全性](https://developers.OpenAI.com/Codex/security) 和[基本配置](https://developers.OpenAI.com/Codex/config-basic) 了解更多详情。

## git

使用Git设置标准化分支命名并选择Codex是否使用强制推送。您还可以设置Codex用于生成提交消息和拉取请求描述的提示词。

## 集成和 MCP通过MCP（模型上下文协议）连接外部工具。启用推荐的服务器或添加您自己的服务器。如果服务器需要 OAuth，应用程序将启动身份验证过程。这些设置也适用于 Codex CLI 和 IDE 扩展，因为 MCP 配置位于 `config.toml` 中。详情请参阅[模型上下文协议文档](https://developers.OpenAI.com/Codex/mcp)。## 个性化

选择**友好**、**务实**或**无**作为您的默认个性。使用 **None** 禁用个性化指令。您可以随时更新此设置。

您还可以添加自己的自定义指令。编辑自定义指令将更新您的[AGENTS.md 中的个人指令](https://developers.OpenAI.com/Codex/guides/agents-md)。

## 已归档的线程**存档的线程**部分列出了已存档的聊天以及日期和项目上下文。使用 **Unarchive** 恢复线程。
