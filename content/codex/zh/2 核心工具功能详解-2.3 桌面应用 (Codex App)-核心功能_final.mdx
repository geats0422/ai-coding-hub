#Codex应用函数

<AdPlaceholder />

Codex 应用程序是一种以桌面为中心的体验，用于并行处理 Codex 线程，具有内置工作树支持、自动化和 Git 功能。

*图片说明：Codex应用功能介绍视频，展示多项目管理、技能选择、自动化创建的界面演示。 *

---

## 跨项目多任务处理

使用 Codex 应用程序窗口跨项目运行任务。为每个代码库添加一个项目并根据需要切换它们。

如果您使用过[Codex CLI](https://developers.OpenAI.com/Codex/cli)，该项目就像在特定目录中启动会话一样。如果您在包含两个或多个应用程序或包的单个存储库中工作，请将不同的项目拆分为单独的应用程序项目，以便 [沙箱](https://developers.OpenAI.com/Codex/security) 仅包含该项目的文件。

*图像描述：Codex 应用程序的界面布局，在侧边栏中显示多个项目，在主面板中显示线程。 *

## 技能支持

Codex 应用程序支持与 CLI 和 IDE 扩展相同的 [代理技能](https://developers.OpenAI.com/Codex/skills)。您还可以通过单击侧栏中的“技能”来查看和探索您的团队在不同项目中创建的技能。

*图像描述：Codex 应用程序中的界面显示可用的技能选择器。 *

---

## 自动化您还可以将技能与 [自动化](https://developers.OpenAI.com/Codex/app/automations) 结合起来执行常见任务，例如评估遥测中的错误并提交修复或创建有关最近代码库更改的报告。

*图片描述：自动创建表单界面，带有时间表和提示词字段。 *

---## 模式

每个线程都以选定的模式运行。启动线程时，您可以选择：

- **本地**：直接在当前项目目录中工作。
- **工作树**：隔离 Git 工作树中的更改。 [了解更多信息](https://developers.OpenAI.com/Codex/app/worktrees)。
- **云**：在配置的云环境中远程运行。

**本地**和**工作树**线程都将在您的计算机上运行。*图像描述：新的线程编辑器显示本地、工作树和云模式选项的界面。 *

有关完整的术语和概念，请参阅[概念部分](https://developers.OpenAI.com/Codex/prompting)。

## 内置 Git 工具

Codex 应用程序在应用程序内提供常见的 Git 功能。

差异面板显示本地项目或工作树签出中更改的 Git 差异。您还可以向 Codex 添加内联注释，并暂存或回滚特定块或整个文件。

您还可以直接从 Codex 应用程序提交、推送和创建本地任务和工作树任务的拉取请求。

对于更高级的 Git 任务，请使用[集成终端](#集成终端)。

*图像描述：Git diff 和带有提交消息字段的提交面板界面。 *

## 工作树支持创建新线程时，选择**本地**或**工作树**。 **本地**直接在您的项目中工作。 **Worktree** 创建一个新的 [Git 工作树](https://git-scm.com/docs/git-worktree)，以便更改与常规项目保持隔离。当您想要尝试新想法而不触及当前工作时，或者当您希望 Codex 在同一项目中并行运行独立任务时，请使用 Worktree。

自动化在 Git 存储库的专用后台工作树中运行，并直接在非版本控制项目的工作目录中运行。

[详细了解如何在 Codex 应用中使用工作树。](https://developers.OpenAI.com/Codex/app/worktrees)

*图片描述：工作树线程视图界面，显示分支操作和工作树详细信息。 *

## 综合终端每个线程都包含一个内置终端，范围仅限于当前项目或工作树。使用终端图标或按 **Cmd**+**J** 打开它。

使用终端验证更改、运行脚本并执行 Git 操作，而无需离开应用程序。

常见任务包括：

-`git status`
-`git pull --rebase`
- `pnpm test` 或 `npm test`
- `pnpm run lint` 或类似的项目命令

如果您定期运行任务，则可以在[本地环境](https://developers.OpenAI.com/Codex/app/local-environments) 中定义**操作**，以将快捷按钮添加到Codex 应用程序窗口的顶部。

<Callout type="note">
**Cmd+K** 打开 Codex 应用程序中的命令选项板。它不会清除终端。要清除终端，请使用 **Ctrl+L**。
</Callout>

## 与 IDE 扩展同步

如果您的编辑器中安装了 [Codex IDE 扩展](https://developers.OpenAI.com/Codex/ide)，则您的 Codex 应用程序和 IDE 扩展在同一项目中时将自动同步。当它们同步时，您将在 Codex 应用程序编辑器中看到 **IDE Con​​text** 选项。启用自动上下文后，Codex 应用程序会跟踪您正在查看的文件，以便您可以间接引用它们（例如，“这个文件是关于什么的？”）。您还可以在 Codex 应用程序的 IDE 扩展中看到运行的线程，反之亦然。

如果您不确定应用程序是否包含上下文，请将其关闭并再次询问相同的问题以比较结果。

## 批准和沙箱

您的批准和沙箱设置限制Codex操作。

- 批准确定 Codex 在运行命令之前何时暂停以获得权限。
- 沙箱控制Codex可以使用哪些目录和网络访问。当您看到“批准一次”或“在此会话中批准”等提示时，您正在向该工具授予不同范围的执行权限。如果您不确定，请批准最窄的选项并继续迭代。

默认情况下，Codex 的工作范围为当前项目。在大多数情况下，这是正确的约束。

如果您的任务需要跨多个存储库或目录进行工作，则更愿意打开单独的项目或使用工作树，而不是要求 Codex 在项目根目录之外漫游。

有关Codex如何处理沙箱的详细信息，请查看[安全文档](https://developers.OpenAI.com/Codex/security)。## MCP 支持Codex 应用程序、CLI 和 IDE 扩展共享 [模型上下文协议 (MCP)](https://developers.OpenAI.com/Codex/mcp) 设置。如果您在其中一个中配置了 MCP 服务器，则它们将自动被另一个中采用。要配置新服务器，请打开应用程序设置中的 MCP 部分并启用推荐的服务器或将新服务器添加到您的配置中。

## 网络搜索Codex 附带官方网络搜索工具。对于 Codex IDE 扩展中的本地任务，Codex 默认启用 Web 搜索并提供来自 Web 搜索缓存的结果。如果您将沙箱配置为[完全访问](https://developers.OpenAI.com/Codex/security)，网络搜索默认为实时结果。请参阅[基本配置](https://developers.OpenAI.com/Codex/config-basic)禁用网络搜索或切换到实时结果以获取最新数据。

## 图片输入

您可以将图像拖放到提示词编辑器中，将它们作为上下文包含在内。按住 **Shift** 的同时拖放图像，将图像添加到上下文中。

您还可以要求Codex查看您系统上的图像。通过向 Codex 提供您正在处理的应用程序的屏幕截图，Codex 可以验证它是否运行良好。## 通知

默认情况下，Codex 应用程序在任务完成或需要批准且应用程序处于后台时发送通知。

在Codex应用程序设置中，您可以选择从不发送通知，或者即使在应用程序处于焦点状态时也始终发送通知。

## 保持计算机处于唤醒状态

由于您的任务可能需要一段时间才能完成，因此您可以通过在应用程序设置中启用“运行时防止睡眠”开关，让 Codex 应用程序阻止您的计算机进入睡眠状态。
