# 评论

审核小组可帮助您了解 Codex 中的更改内容、提供有针对性的反馈并决定保留哪些内容。

它仅适用于位于 Git 存储库中的项目。如果您的项目还不是 Git 存储库，审阅面板将提示您创建一个。

<AdPlaceholder />

## 显示的变化

审核面板反映了 Git 存储库的状态，而不仅仅是 Codex 编辑的内容。这意味着它将显示：

- Codex 变化
- 你自己的改变
- 存储库中任何其他未提交的更改

默认情况下，审核小组重点关注未提交的更改。您还可以将范围切换到：

- **所有分支更改**（与基础分支的差异）
- **最后一轮变化**（仅限最新助理轮次）在本地工作时，您还可以在 **Unstaged** 和 **Staging** 更改之间切换。

## 浏览评论面板

- 单击文件名通常会在您选择的编辑器中打开该文件。您可以在[设置](https://developers.OpenAI.com/Codex/settings)中选择默认编辑器。
- 单击文件名背景可展开或折叠差异。
- 按住 **Cmd** 的同时单击单行将在选择编辑器中打开该行。
- 如果您对更改感到满意，可以[暂存更改或回滚不喜欢的更改](#暂存和回滚文件)。

## 内嵌评论以获取反馈

内联注释允许您将反馈直接附加到差异中的特定行。这通常是指导 Codex 进行正确修复的最快方法。

要留下内嵌评论：1. 打开审阅面板。
2. 将鼠标悬停在您要发表评论的行上。
3. 单击出现的 **+** 按钮。
4. 写下您的反馈并提交。
5.所有反馈完成后，向线程发回消息。

由于注释固定在一行上，Codex 通常可以比一般指令更准确地做出响应。

内联评论被视为审阅指南。留下评论后，发送一条意图明确的后续消息，例如“内嵌评论并将范围保持在最低限度”。

##暂存和回滚文件审核面板包括 Git 操作，因此您可以在提交之前调整差异。

您可以在多个级别暂存、取消暂存或回滚更改：

- **整个差异**：使用审阅标题中的操作按钮（例如“全部暂存”或“全部回滚”）
- **每个文件**：暂存、取消暂存或回滚单个文件
- **每块**：暂存、取消暂存或回滚单个块

当您想要接受部分工作时使用暂存，当您想要放弃它时使用回滚。

### 部分临时状态

Git 可以表示同一文件中的暂存和未暂存更改。发生这种情况时，面板似乎在暂存和未暂存视图中“显示同一文件两次”。这是正常的 Git 行为。
