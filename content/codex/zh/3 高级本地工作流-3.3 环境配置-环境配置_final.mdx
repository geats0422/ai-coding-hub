# 本地环境

本地环境允许您配置项目的工作树设置步骤和常用操作。

您可以通过[Codex 应用设置](Codex://settings) 窗格配置本地环境。您可以将生成的文件签入项目的 Git 存储库中，以便与其他人共享。

Codex将此配置存储在项目根目录的`.Codex`文件夹中。如果您的存储库包含多个项目，请打开包含共享`.Codex` 文件夹的项目目录。

<AdPlaceholder />

## 设置脚本由于工作树与本地任务在不同的目录中运行，因此您的项目可能未完全设置，并且可能缺少未检入存储库的依赖项或文件。设置脚本在 Codex 创建新工作树时在新线程开始时自动运行。

使用此脚本运行配置环境所需的任何命令，例如安装依赖项或运行构建过程。

例如，对于 TypeScript 项目，您可能希望使用安装脚本来安装依赖项并执行初始构建：
```
bas
h
npm install
npm run build
```
如果您的设置是特定于平台的，请为 macOS、Windows 或 Linux 定义设置脚本以覆盖默认值。
