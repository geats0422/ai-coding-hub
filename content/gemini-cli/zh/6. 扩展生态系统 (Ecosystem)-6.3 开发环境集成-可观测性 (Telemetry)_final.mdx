## 主要优点

- **🔍 使用情况分析**：了解交互模式和功能采用情况
  整个团队
- **⚡ 性能监控**：跟踪响应时间、令牌消耗和
  资源利用
- **🐛 实时调试**：识别瓶颈、故障和错误模式
  当它们发生时
- **📊 工作流程优化**：做出明智的决策以改进
  配置和流程
- **🏢 企业治理**：监控跨团队的使用情况，跟踪成本，确保
  合规性，并与现有监控基础设施集成

## OpenTelemetry 集成

基于 **[OpenTelemetry]** 构建 - 供应商中立的行业标准
可观察性框架 — Gemini CLI 的可观察性系统提供：- **通用兼容性**：导出到任何 OpenTelemetry 后端（Google
  Cloud、Jaeger、Prometheus、Datadog 等）
- **标准化数据**：使用一致的格式和收集方法
  你的工具链
- **面向未来的集成**：与现有和未来的可观察性相连接
  基础设施
- **无供应商锁定**：在后端之间切换而无需更改您的
  仪器仪表

[开放遥测]：https://opentelemetry.io/

## 配置

所有遥测行为均通过您的`.Gemini/settings.json` 文件控制。
环境变量可用于覆盖文件中的设置。|设置|环境变量|描述 |价值观 |默认|
| -------------- | -------------------------------- | --------------------------------------------------- | ----------------- | ----------------------- |
| `enabled` | `GEMINI_TELEMETRY_ENABLED` |启用或禁用遥测 | `true`/`false`| `false` |
| `target` | `GEMINI_TELEMETRY_TARGET` |向何处发送遥测数据 | `"gcp"`/`"local"` | `"local"` |
| `otlpEndpoint` | `GEMINI_TELEMETRY_OTLP_ENDPOINT` | OTLP 收集器端点 |网址字符串| `http://localhost:4317` |
| `otlpProtocol` | `GEMINI_TELEMETRY_OTLP_PROTOCOL` | OTLP传输协议| `"grpc"`/`"http"` | `"grpc"` |
| `outfile` | `GEMINI_TELEMETRY_OUTFILE` |将遥测数据保存到文件（覆盖`otlpEndpoint`）|文件路径 | - |
| `logPrompts` | `GEMINI_TELEMETRY_LOG_PROMPTS` |在遥测日志中包含提示 | `true`/`false`| `true` |
| `useCollector` | `GEMINI_TELEMETRY_USE_COLLECTOR` |使用外部 OTLP 收集器（高级）| `true`/`false` | `false` |
| `useCliAuth` | `GEMINI_TELEMETRY_USE_CLI_AUTH` |使用 CLI 凭据进行遥测（仅限 GCP 目标）| `true`/`false` | `false` |
<Callout type="note">
关于布尔环境变量的注意事项：对于布尔设置（`enabled`，
`logPrompts`,`useCollector`)，设置相应的环境变量为
`true` 或 `1` 将启用该功能。任何其他值都会禁用它。
</Callout>

有关所有配置选项的详细信息，请参阅
[Configuration guide](/docs/gemini/cli-settings-preferences)。

## Google Cloud 遥测

### 先决条件

在使用以下任一方法之前，请完成以下步骤：

1. 设置您的 Google Cloud 项目 ID：
   - 对于来自推理的单独项目中的遥测：
```
bas
h
     export OTLP_GOOGLE_CLOUD_PROJECT="your-telemetry-project-id"
     
```
-
 对于与推理相同的项目中的遥测：
```
bas
h
     export GOOGLE_CLOUD_PROJECT="your-project-id"
     
```
2
. 使用 Google Cloud 进行身份验证：
   - 如果使用用户帐户：
```
bas
h
     gcloud auth application-default login
     
```
-
 如果使用服务帐户：
```
bas
h
     export GOOGLE_APPLICATION_CREDENTIALS="/path/to/your/service-account.json"
     
```
3
. 确保您的账户或服务账户具有以下 IAM 角色：
   - 云追踪代理
   - 监控指标编写器
   - 日志编写器

4. 启用所需的 Google Cloud API（如果尚未启用）：
```
bas
h
   gcloud services enable \
     cloudtrace.googleapis.com \
     monitoring.googleapis.com \
     logging.googleapis.com \
     --project="$OTLP_GOOGLE_CLOUD_PROJECT"
   
```
### 使用 CLI 凭证进行身份验证

默认情况下，Google Cloud 的遥测收集器使用应用程序默认值
凭证 (ADC)。但是，您可以将其配置为使用相同的 OAuth
您用于登录Gemini CLI 的凭据。这在以下方面很有用
没有设置 ADC 的环境。

要启用此功能，请将 `telemetry` 设置中的 `useCliAuth` 属性设置为
`true`：
```
jso
n
{
  "telemetry": {
    "enabled": true,
    "target": "gcp",
    "useCliAuth": true
  }
}
```
<Callout type="warning">
重要：

- 此设置需要使用**直接导出**（进程内导出器）。
- 它**不能**与`useCollector: true`一起使用。如果同时启用两者，遥测
  将被禁用并记录错误。
- CLI 将自动使用您的凭据通过 Google 进行身份验证
  Cloud Trace、指标和日志记录 API。
</Callout>

### 直接导出（推荐）

将遥测数据直接发送到 Google Cloud 服务。无需收集器。

1. 在`.Gemini/settings.json`中启用遥测：
```
jso
n
   {
     "telemetry": {
       "enabled": true,
       "target": "gcp"
     }
   }
   
```
2
. 运行Gemini CLI 并发送提示。
3.查看日志和指标：
   - 发送提示后在浏览器中打开 Google Cloud Console：
     - 日志：https://console.cloud.google.com/logs/
     - 指标：https://console.cloud.google.com/monitoring/metrics-explorer
     - 痕迹：https://console.cloud.google.com/traces/list

### 基于收集器的导出（高级）

对于自定义处理、过滤或路由，请使用 OpenTelemetry 收集器
将数据转发到 Google Cloud。

1. 配置您的`.Gemini/settings.json`：
```
jso
n
   {
     "telemetry": {
       "enabled": true,
       "target": "gcp",
       "useCollector": true
     }
   }
   
```
2
. 运行自动化脚本：
```
bas
h
   npm run telemetry -- --target=gcp
   
```
这将：
   - 启动转发到 Google Cloud 的本地 OTEL 收集器
   - 配置您的工作空间
   - 提供在 Google Cloud Console 中查看跟踪记录、指标和日志的链接
   - 将收集器日志保存到`~/.Gemini/tmp/<projectHash>/otel/collector-gcp.log`
   - 退出时停止收集器（例如`Ctrl+C`）
3. 运行Gemini CLI 并发送提示。
4.查看日志和指标：
   - 发送提示后在浏览器中打开 Google Cloud Console：
     - 日志：https://console.cloud.google.com/logs/
     - 指标：https://console.cloud.google.com/monitoring/metrics-explorer
     - 痕迹：https://console.cloud.google.com/traces/list
   - 打开`~/.Gemini/tmp/<projectHash>/otel/collector-gcp.log`查看本地
     收集器日志。

## 本地遥测对于本地开发和调试，您可以在本地捕获遥测数据：

### 基于文件的输出（推荐）

1. 在`.Gemini/settings.json`中启用遥测：
```
jso
n
   {
     "telemetry": {
       "enabled": true,
       "target": "local",
       "otlpEndpoint": "",
       "outfile": ".gemini/telemetry.log"
     }
   }
   
```
2
. 运行Gemini CLI 并发送提示。
3. 查看指定文件（例如`.Gemini/telemetry.log`）中的日志和指标。

### 基于收集器的导出（高级）

1. 运行自动化脚本：
```
bas
h
   npm run telemetry -- --target=local
   
```
这将：
   - 下载并启动 Jaeger 和 OTEL 收集器
   - 配置您的工作区以进行本地遥测
   - 在 http://localhost:16686 提供 Jaeger UI
   - 将日志/指标保存到`~/.Gemini/tmp/<projectHash>/otel/collector.log`
   - 退出时停止收集器（例如`Ctrl+C`）
2. 运行Gemini CLI 并发送提示。
3. 查看 http://localhost:16686 处的跟踪以及收集器日志中的日志/指标
   文件。

## 日志和指标

以下部分描述了生成的日志和指标的结构
Gemini CLI。

`session.id`、`installation.id` 和`user.email`（仅当
通过 Google 帐户进行身份验证）作为公共属性包含在所有
日志和指标。

### 日志日志是特定事件的带时间戳的记录。记录以下事件
对于Gemini CLI，按类别分组。

#### 会议

捕获启动配置和用户提示提交。- `gemini_cli.config`：使用 CLI 配置启动时发出一次。
  - **属性**：
    - `model`（字符串）
    - `embedding_model`（字符串）
    - `sandbox_enabled`（布尔值）
    - `core_tools_enabled`（字符串）
    - `approval_mode`（字符串）
    - `api_key_enabled`（布尔值）
    - `vertex_ai_enabled`（布尔值）
    - `log_user_prompts_enabled`（布尔值）
    - `file_filtering_respect_git_ignore`（布尔值）
    - `debug_mode`（布尔值）
    - `mcp_servers`（字符串）
    - `mcp_servers_count` (int)
    - `extensions`（字符串）
    - `extension_ids`（字符串）
    - `extension_count` (int)
    - `mcp_tools`（字符串，如果适用）
    - `mcp_tools_count`（整数，如果适用）
    - `output_format`（“文本”、“json”或“stream-json”）- `gemini_cli.user_prompt`：当用户提交提示时发出。
  - **属性**：
    - `prompt_length` (int)
    - `prompt_id`（字符串）
    - `prompt`（字符串；如果`telemetry.logPrompts` 是`false`，则排除）
    - `auth_type`（字符串）

#### 审批模式

跟踪批准模式的变化和持续时间。

##### 生命周期

- `approval_mode_switch`：批准模式已更改。
  - **属性**：
    - `from_mode`（字符串）
    - `to_mode`（字符串）

- `approval_mode_duration`：处于批准模式的持续时间。
  - **属性**：
    - `mode`（字符串）
    - `duration_ms` (int)

##### 执行

这些事件跟踪批准模式（例如计划模式）的执行情况。- `plan_execution`：执行了计划并且会话从计划模式切换
  到主动执行。
  - **属性**：
    - `approval_mode`（字符串）

#### 工具

捕获工具执行、输出截断和编辑行为。- `gemini_cli.tool_call`：为每个工具（函数）调用发出。
  - **属性**：
    -`function_name`
    -`function_args`
    -`duration_ms`
    - `success`（布尔值）
    - `decision`（“接受”、“拒绝”、“自动接受”或“修改”，如果适用）
    - `error`（如果适用）
    - `error_type`（如果适用）
    - `prompt_id`（字符串）
    - `tool_type`（“本机”或“mcp”）
    - `mcp_server_name`（字符串，如果适用）
    - `extension_name`（字符串，如果适用）
    - `extension_id`（字符串，如果适用）
    - `content_length`（整数，如果适用）
    - `metadata`（如果适用）- `gemini_cli.tool_output_truncated`：工具调用的输出被截断。
  - **属性**：
    - `tool_name`（字符串）
    - `original_content_length` (int)
    - `truncated_content_length` (int)
    - `threshold` (int)
    - `lines` (int)
    - `prompt_id`（字符串）

- `gemini_cli.edit_strategy`：编辑所选策略。
  - **属性**：
    - `strategy`（字符串）

- `gemini_cli.edit_correction`：编辑校正结果。
  - **属性**：
    - `correction`（“成功”|“失败”）-`gen_ai.client.inference.operation.details`：本次活动提供详细
  有关 GenAI 操作的信息，与 [OpenTelemetry GenAI
  事件的语义约定]。
  - **属性**：
    - `gen_ai.request.model`（字符串）
    - `gen_ai.provider.name`（字符串）
    - `gen_ai.operation.name`（字符串）
    - `gen_ai.input.messages`（json 字符串）
    - `gen_ai.output.messages`（json 字符串）

- `gemini_cli.web_fetch_fallback_attempt`：尝试网络获取回退。
  - **属性**：

#### 扩展跟踪扩展生命周期和设置更改。

  - **属性**：

    - `setting_scope`（字符串）

  - **属性**：

    - `setting_scope`（字符串）- `gemini_cli.extension_update`：扩展已更新。
  - **属性**：

    - `extension_version`（字符串）
