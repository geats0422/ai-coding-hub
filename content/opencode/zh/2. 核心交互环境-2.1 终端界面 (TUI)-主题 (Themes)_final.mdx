通过 OpenCode，你可以从几个内置主题中选择，使用适你终端主题的主题，或定义自己的自定义主题。

默认情况下，OpenCode 使用我们自己的 opencode 主题。

## 终端要求

为了使主题以其完整的调色板正确显示，你的终端必须支持真彩色（24 位颜色）。大多数现代终端默认支持这一点，但可能需要启用它：

- 检查支持：运行 `echo $COLORTERM` - 它应该输出 truecolor 或 24bit
- 启用真彩色：在你的 shell 配置文件中设置环境变量 `COLORTERM=truecolor`
- 终端兼容性：确保你的终端模拟器支持 24 位颜色（大多数现代终端如 iTerm2、Alacritty、Kitty、Windows Terminal 和 GNOME Terminal 的最新版本都支持）

如果不支持真彩色，主题可能会以降低的颜色准确性显示或回退到最近的 256 色近似。

## 系统主题

系统主题设计为自动适应终端的配色方案。与使用固定颜色的传统主题不同，系统主题：

- 生成灰度：根据终端的背景颜色创建自定义灰度，确保最佳对比度
- 使用 ANSI 颜色：使用标准 ANSI 颜色（0-15）进行语法高亮和 UI 元素，这些颜色尊重终端的调色板
- 保留终端默认：使用 none 表示文本和背景颜色以维护终端的原生外观

系统主题适用于：

- 希望 OpenCode 匹配终端的外观
- 使用自定义终端配色方案
- 偏好所有终端应用之间的一致外观

## 使用主题

你可以通过使用 /theme 命令调出主题选择来选择主题。或者你可以在配置中指定它。

```json
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "tokyonight"
}
```

## 自定义主题

OpenCode 支持灵活的基于 JSON 的主题系统，允许用户轻松创建和自定义主题。

### 层次结构

主题从多个目录按以下顺序加载，后面的目录覆盖前面的目录：

- 内置主题 - 这些嵌入在二进制文件中
- 用户配置目录 - 在 ~/.config/opencode/themes/*.json 或 $XDG_CONFIG_HOME/opencode/themes/*.json 中定义
- 项目根目录 - 在 `<project-root>/.opencode/themes/*.json` 中定义
- 当前工作目录 - 在 ./.opencode/themes/*.json 中定义

如果多个目录包含同名的主题，则使用来自较高优先级目录的主题。

### 创建主题

要创建自定义主题，请在某个主题目录中创建一个 JSON 文件。

对于用户范围的的主题：

```bash
mkdir -p ~/.config/opencode/themes
vim ~/.config/opencode/themes/my-theme.json
```

对于项目特定的主题：

```bash
mkdir -p .opencode/themes
vim .opencode/themes/my-theme.json
```

## JSON 格式

主题使用灵活的 JSON 格式，支持：

- 十六进制颜色："#ffffff"
- ANSI 颜色：3（0-255）
- 颜色引用："primary" 或自定义定义
- 深色/浅色变体：`{"dark": "#000"，"light": "#fff"}`
- 无颜色："none" - 使用终端的默认颜色或透明

### 颜色定义

defs 部分是可选的，它允许你在主题中定义可重用的颜色。

### 终端默认

特殊值 "none" 可用于任何颜色以继承终端的默认颜色。这对于创建与终端配色方案无缝融合的主题特别有用：

- `"text": "none"` - 使用终端的默认前景颜色
- `"background": "none"` - 使用终端的默认背景颜色
