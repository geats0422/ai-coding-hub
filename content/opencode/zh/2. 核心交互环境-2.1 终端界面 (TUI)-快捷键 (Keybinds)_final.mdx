## 引导键

OpenCode 使用引导键来进行大多数快捷键绑定。这避免了你的终端中的冲突。

默认情况下，`ctrl+x` 是引导键，大多数操作需要你先按引导键，然后按快捷键。例如，要开始一个新会话，你先按 `ctrl+x`，然后按 `n`。

你不需要为你的快捷键使用引导键，但我们推荐这样做。

## 禁用快捷键

你可以通过在配置中添加一个值为"none"的键来禁用快捷键。

```json
{
  "$schema": "https://opencode.ai/config.json",
  "keybinds": {
    "session_compact": "none"
  }
}
```

## 桌面提示快捷键

OpenCode 桌面应用提示输入支持常见的 Readline/Emacs 风格快捷键用于编辑文本。这些是内置的，目前无法通过 opencode.json 配置。

| 快捷键 | 操作 |
|----------|--------|
| `ctrl+a` | 移动到当前行的开头 |
| `ctrl+e` | 移动到当前行的末尾 |
| `ctrl+b` | 光标向后移动一个字符 |
| `ctrl+f` | 光标向前移动一个字符 |
| `alt+b` | 光标向后移动一个单词 |
| `alt+f` | 光标向前移动一个单词 |
| `ctrl+d` | 删除光标下的字符 |
| `ctrl+k` | 删除到行尾 |
| `ctrl+u` | 删除到行首 |
| `ctrl+w` | 删除前一个单词 |
| `alt+d` | 删除下一个单词 |
| `ctrl+t` | 调换字符 |
| `ctrl+g` | 取消弹出窗口 / 中止正在运行的响应 |

<Callout type="note">
Shift+Enter：某些终端默认不发送带 Enter 的修饰键。你可能需要配置终端将 Shift+Enter 作为转义序列发送。
</Callout>

### Windows Terminal

在以下位置打开你的 settings.json：

`%LOCALAPPDATA%\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json`

将此添加到根级操作数组：

```json
{
  "actions": [
    {
      "command": {
        "action": "sendInput",
        "input": "\u001b[13;2u"
      },
      "id": "User.sendInput.ShiftEnterCustom"
    }
  ]
}
```

将此添加到根级键绑定数组：

```json
{
  "keybindings": [
    {
      "keys": "shift+enter",
      "id": "User.sendInput.ShiftEnterCustom"
    }
  ]
}
```

保存文件并重启 Windows Terminal 或打开一个新标签。
