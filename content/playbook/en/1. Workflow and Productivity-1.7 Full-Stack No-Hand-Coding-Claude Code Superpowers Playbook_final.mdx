## 0x01. The Paradigm Shift (Features)

The core of `Claude Code + Superpowers` is not "a few more commands." It is upgrading the workflow from improvisational coding into a hard protocol: design first, plan second, execute last.

- **Brainstorm first**: compress vague requirements into verifiable design documents and data models before implementation.
- **Plan decomposition**: break the work into 20+ executable tasks with explicit files, steps, and verification commands.
- **Subagent parallel execution**: keep the main session on spec control while subagents execute in batches and self-check with `npm run build`.

> Notes
> The exact command shape for `Superpowers` may vary across hosts or versions, such as `/superpowers:brainstorm`, `/superpowers brainstorm`, or skill-based triggers. Confirm syntax against your current version docs (**needs verification**).

## 0x04. The Workflow

### Scenario: from vague requirement to a shippable full-stack app

Goal: build a Next.js + Supabase app with complex API authorization and keep manual coding close to zero.

```text
# Step A: architecture design (adjust command shape by version)
/superpowers brainstorm

Requirement: build a D1/Supabase data-layer app with complex API authorization.
Extra constraints: clarify auth boundaries, failure paths, audit fields, and rollback strategy first. If anything is unclear, keep asking instead of writing code.
Output: project design doc + database Schema + draft auth strategy.
```

```text
# Step B: task dispatch
/superpowers write-plan

Lock boundaries first.
```

```text
# Step C: iterative execution
/superpowers execute-plan

Execute in batches and return changed files, verification results, and failure reasons every round.
```

```bash
# Mandatory self-check after each batch
npm run build
```

```text
Expected:
- You get a traceable task chain instead of one giant code dump
- Subagents move in parallel while the main session handles spec alignment and acceptance
- Every round ends with reproducible build validation
```

## 0x05. Troubleshooting

- Error: token overflow in large projects suddenly degrades output quality
  - Fix:

```text
/compact "Keep only implementation plan, changed files, and last error"
/clear
```

```text
Split execute-plan into smaller batches of 3-5 tasks.
```

- Error: blank page during early UI rendering because route state is out of sync
  - Fix:

```text
Send screenshots, console errors, and current route state to the agent in one bundle.
```

```bash
npm run build
```

```text
Require the agent to return: root cause, changed files, and verification steps.
```

- Error: persistence layer never actually takes effect because Schema was generated but not applied
  - Fix:

```text
Paste the generated Schema into Supabase SQL Editor and execute it manually.
```
