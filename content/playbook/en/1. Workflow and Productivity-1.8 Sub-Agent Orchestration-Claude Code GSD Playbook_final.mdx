## 0x01. The Paradigm Shift (Features)

The core of `Claude Code + GSD` is not "a few more commands." It is transforming the delivery process into an orchestratable system.

- **Sub-agent context isolation**: Planning, development, and verification are delegated to different agents. The main session only handles scheduling and decision-making.
- **Wave execution**: Parallelizable tasks run in parallel; dependent tasks advance sequentially and automatically.
- **Verification-driven delivery**: Every phase comes with verification reports and traceable commits, not just "looks about right."

## 0x04. The Workflow

### Scenario 1: Cold start for existing projects (map first, then build)

Goal: Let agents understand the codebase before breaking down requirements.

```text

```

```text
/gsd:new-project
```

```text

- Requirements are tiered into v1/v2/out-of-scope
- Roadmap can directly enter phase execution
```

### Scenario 2: Sequential dependency chains with wave-based parallelism

Goal: Land critical dependencies in order while parallelizing independent tasks via wave execution.

```text
/gsd:plan-phase 1
/gsd:execute-phase 1
/gsd:verify-work 1
```

```text
Expected:
- Critical dependencies land in order; independent tasks run in waves
- Each phase ends with verification conclusions and traceable commits
```

## 0x05. Troubleshooting

- Error: Aggressive parallelism triggers rate limiting or quota pressure, tasks stall "half-complete"
  - Fix:

```text
Prioritize sequential execution for critical phases, then parallelize non-dependent tasks
For 429 errors, integrate Retry-After headers with exponential backoff
Cap concurrent agent count to avoid peak request bursts
```

- Error: Zustand state updates do not refresh UI or cause abnormal re-renders
  - Fix:

```typescript
import { useShallow } from 'zustand/react/shallow'
const { a, b } = useStore(useShallow((s) => ({ a: s.a, b: s.b })))
```

```text
Avoid whole-store subscriptions; prefer fine-grained selectors to reduce unnecessary re-renders
```
