## 0x01. The Paradigm Shift (破局之道 / Features)

`Claude Code + Superpowers` 的核心，不是“多几个命令”，而是把工作流从“想到哪写到哪”升级成“先设计、再计划、后执行”的硬协议。

- **头脑风暴先行**：先把模糊需求压成可验证的设计文档与数据模型，减少后续返工。
- **计划拆解落地**：把大需求拆到 20+ 可执行任务，明确文件、步骤、验证命令。
- **子代理并行执行**：主会话守规格，子代理分批施工，配合 `npm run build` 快速自检。

> 注意事项
> `Superpowers` 的具体命令形态在不同宿主或版本可能存在差异（例如 `/superpowers:brainstorm`、`/superpowers brainstorm`，或通过 skill 触发）。命令拼写请以你当前版本文档为准（**待验证**）。

## 0x04. The Workflow (实战推演)

### 场景：从模糊需求到可发布全栈应用

目标：构建“带复杂 API 鉴权”的 Next.js + Supabase 应用，尽量零手写。

```text
# Step A: 架构设计（命令形态按版本调整）
/superpowers brainstorm

需求：构建带复杂 API 鉴权的 D1/Supabase 数据层应用。
补充约束：先把鉴权边界、失败路径、审计字段、回滚策略一次问清，不清楚就继续追问，不要直接开写。
输出：项目设计文档 + 数据库 Schema + 鉴权策略草案。
```

```text
# Step B: 任务分发
/superpowers write-plan

先定边界。
```

```text
# Step C: 循环迭代执行
/superpowers execute-plan

按批次执行，每轮都回传：改动文件、验证结果、失败原因。
```

```bash
# 每批次执行后强制自检
npm run build
```

```text
Expected:
- 你得到可追踪任务链，而不是一次性大段代码
- 子代理并行推进，主会话只做规格对齐与验收
- 每轮都有可复现的构建验证结果
```

## 0x05. 避坑指南 (Troubleshooting)

- Error: 巨型项目触发 Token 溢出，会话输出突然劣化
  - Fix:

```text
/compact "Keep only implementation plan, changed files, and last error"
/clear
```

```text
将 execute-plan 拆成更小批次（每批 3-5 个任务）
```

- Error: UI 渲染初期出现空白页（路由状态不同步）
  - Fix:

```text
截图 + 控制台报错 + 当前路由状态，一次性回传给 Agent
```

```bash
npm run build
```

```text
要求 Agent 返回：问题根因、改动文件、验证步骤
```

- Error: 持久化层未生效（Schema 仅生成未落库）
  - Fix:

```text
把 Agent 生成的 Schema 粘贴到 Supabase SQL Editor 手动执行
```
