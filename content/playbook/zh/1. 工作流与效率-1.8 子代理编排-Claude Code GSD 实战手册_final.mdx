## 0x00. The Matrix (陈述乱象 / Agitate)

你可能每天都在重复同一套低效动作：

- 一个会话里同时塞需求、修 bug、写测试、看日志
- 提示词越写越长，但结果越来越漂
- 出错后临时补丁，不做验证闭环
- 每轮都从头解释约束，主会话上下文持续脏化

这不是"还不够努力"。

这是上下文饱和后的精度衰减：你以为在推进，实际上在交上下文税。

## 0x01. The Paradigm Shift (破局之道 / Features)

`Claude Code + GSD` 的核心，不是"多几个命令"，而是把交付流程变成可编排系统。

- **子代理（Subagent）隔离上下文（Context）**：规划、研发、验证拆给不同代理执行，主会话只做调度与决策。
- **波次执行（wave execution）**：可并行任务并行跑，有依赖的任务自动顺序推进。
- **验证驱动交付**：每个阶段都带验证报告和可追溯提交，而不是"看起来差不多"。

## 0x04. The Workflow (实战推演)

### 场景 1：存量项目冷启动（先建地图，再开工）

目标：让代理先理解代码库，再做需求拆解。

```text
/gsd:map-codebase
```

```text
/gsd:new-project
```

```text
Expected:
- 生成代码库映射与约束上下文
- 需求被分层到 v1/v2/out-of-scope
- Roadmap 可直接进入阶段（phase）执行
```

### 场景 2：依赖链顺序推进，独立任务按波次并行

目标：让关键依赖按顺序落地，同时把可并行部分留给 wave execution。

```text
/gsd:plan-phase 1
/gsd:execute-phase 1
/gsd:verify-work 1
```

```text
Expected:
- 关键依赖按顺序落地，独立任务按波次并行
- 阶段完成后有验证结论与可回溯提交
```

## 0x05. 避坑指南 (Troubleshooting)

- Error: 并行开太猛，触发限流/额度压力，任务停在"半完成"
  - Fix:

```text
优先顺序执行关键阶段（phase），再并行非依赖任务
429 场景接入 Retry-After + 指数退避
控制并发代理数，避免同一时刻高峰请求
```

- Error: Zustand 状态更新后 UI 不刷新或异常重渲染
  - Fix:

```typescript
import { useShallow } from 'zustand/react/shallow'
const { a, b } = useStore(useShallow((s) => ({ a: s.a, b: s.b })))
```

```text
避免整仓订阅；优先细粒度 selector，减少无效重渲染
```

## 0x06. The Ascension (飞升 / 下一步行动)

别再让一个会话承担所有工作。

把任务拆成"规划-执行-验证"三段，把上下文交给子代理隔离，把结果交给验证协议闭环。

今天就做一次：先 `/gsd:map-codebase`，再跑一轮完整阶段（phase），把"感觉完成"变成"证据完成"。
